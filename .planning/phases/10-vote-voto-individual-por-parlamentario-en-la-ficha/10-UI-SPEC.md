## 3. Page Shell & VOTE Section Layout

### 3.0 Route & shell — `/parlamentario/[id]`

Single-column Server Component. `[id]` is `parlamentario.id`; validate it before any DB read
(numeric/uuid guard, mirroring `BOLETIN_RE` discipline in `/proyecto/[boletin]`). Profile page
renders ONLY for a parlamentario that exists in the maestra; otherwise `notFound()`.

```
<main className="max-w-3xl mx-auto px-4 md:px-8 py-8 md:py-16">
  <Suspense fallback={<ParlamentarioHeaderSkeleton />}>
    <ParlamentarioHeader id={id} />          {/* §3.1 — the reusable shell */}
  </Suspense>

  <section id="votos" className="mt-12">
    <h2 className="text-xl font-semibold mb-4">Votaciones</h2>
    <Suspense fallback={<VotosSkeleton />}>
      <VotosSection id={id} searchParams={...} />   {/* §3.2 */}
    </Suspense>
  </section>

  {/* Phase 11+ append <section id="lobby">, <section id="patrimonio">, ... here.
      Each its own <h2> + Suspense + honest empty state. Do NOT nest into #votos. */}
</main>
```

The VOTE section itself contains, in order: attendance summary (§3.3) → vote list grouped by
votación (§3.2) → Voto×tema facet (§3.4) → Rebeldías (§3.5). Each is a sub-block with its own
`<h3>` and its own empty state.

### 3.1 ParlamentarioHeader (the reusable shell — built this phase, reused by 11–16)

```
[ CamaraChip ]  [ bancada/partido chip ]            ← row, gap-2, flex-wrap
[ Nombre completo ]                                  ← text-3xl font-semibold leading-tight, mt-4
[ cargo + circunscripción/distrito ]                 ← text-sm text-muted-foreground, mt-1
[ photo optional ]                                   ← see rules below
[ ProvenanceBadge ]                                  ← mt-4, fuente de la maestra (Cámara/Senado)
```

- **CamaraChip:** reuse verbatim. Diputado → "Cámara", senador → "Senado".
- **Bancada/partido chip:** shadcn `Badge variant="secondary" className="font-normal"`, label =
  partido o bancada literal de la maestra. This is **descriptive metadata only** — never colored
  by party, never used to rank. Reuses the iniciativa/materia chip pattern from `FichaHeader`.
- **Photo (optional):** if a sourced photo URL exists, render a `rounded-full w-16 h-16 object-cover`
  avatar with `alt="{nombre}"`. If absent → render nothing (no placeholder silhouette that could
  read as a mugshot). Never block the page on a missing photo.
- **ProvenanceBadge:** mandatory. `sourceName` from `sourceLabel(origen)`, `capturedAt` =
  `fecha_captura` of the maestra row, `sourceUrl` = official profile link if present (else null →
  badge still shows, no link — never omit it).
- **Heading hierarchy:** `<h1>` = nombre del parlamentario (one per page).

### 3.2 Vote list — grouped by votación/fecha, paginated

The per-legislator inversion of v1.0's per-bill votación. Each row = one vote the legislator
cast on one votación.

**Grouping:** votes are grouped by votación, ordered by `fecha DESC` (most recent first). Group
header carries the votación context so the row stays legible:

```
<article>  (one per votación)
  <header>
    [ CamaraChip ]  [ fecha — font-mono text-sm text-muted-foreground ]    ← row, gap-2
    [ proyecto título o etapa — text-sm ]
  </header>
  <VotoFichaRow />   ← the legislator's option on THIS votación (see below)
</article>
```

**VotoFichaRow anatomy** (a sibling of v1.0 `VotoRow`, oriented to the legislator's own ficha):

```
[ opción chip ]   [ "Boletín N°XXXX-XX" ]   [ ProvenanceBadge ]
```

- **Opción chip:** reuse `SELECCION_STYLE` from `voto-row.tsx`, **extended with `ausente`**:

  | Seleccion | Label | className |
  |-----------|-------|-----------|
  | `si` | "A favor" | `bg-green-100 text-green-800` |
  | `no` | "En contra" | `bg-red-100 text-red-800` |
  | `abstencion` | "Abstención" | `bg-amber-100 text-amber-800` |
  | `pareo` | "Pareo" | `bg-slate-100 text-slate-600` |
  | `ausente` | "Ausente" | `bg-slate-100 text-slate-500` |

  Map all official enums faithfully — **never collapse `abstencion`/`ausente`/`pareo` into "no
  votó"** (v1.0 fidelity rule). The Seleccion type must be extended to include `"ausente"`.

- **Boletín:** `font-mono`, links to `/proyecto/${boletin}` (an internal, already-trusted route —
  the project exists; this is NOT the identity guard, which governs the *legislator* link).
- **ProvenanceBadge:** per row. `sourceName="Cámara"` (opendata.camara.cl), `capturedAt` =
  row `fecha_captura`, `sourceUrl` = the source votación record.

**Pagination:** server-driven via `?votosPage=N` (or cursor on `fecha`). Page size **20 votaciones**.
Controls: "Anteriores" / "Siguientes" anchors (`min-h-[44px]`), plus "Página N de M" text. No
infinite scroll (press/citizen workflow needs stable, linkable pages). Pagination must be SSR so a
journalist can deep-link a page.

**Identity guard at the page level (LOCKED, VOTE-02, riesgo #1):** the profile page itself is a
strong identity claim. It renders only for a `parlamentario` whose votes are linked by
`estado_vinculo='confirmado'` (Cámara DIPID→id_diputado_camara, deterministic). For any vote whose
link is NOT confirmed, see §3.6 state (c) — it is **never** silently attributed to this profile.

### 3.3 Attendance summary (asistencia)

A descriptive aggregate at the top of the VOTE section, above the list.

```
<h3 className="text-sm font-semibold">Asistencia</h3>
[ CSS-pure bar: votos emitidos vs votaciones en que era miembro ]
[ "Emitió N de M votaciones registradas (legislatura vigente)" — text-sm text-muted-foreground ]
[ desglose: "A favor N · En contra N · Abstención N · Pareo N · Ausente N" ]
```

- **Bar:** reuse the `VotacionBar` CSS pattern (single `flex` row of colored `div`s, `role="img"`,
  per-segment `aria-label`). **No information by color alone** — every count repeated in the text
  row below (a11y, v1.0 §8).
- **Honest denominator:** M = divisiones en que el parlamentario era miembro de la sala/comisión en
  esa fecha, NOT total nationwide. If the denominator cannot be computed honestly, show the raw
  numerator only ("Emitió N votos registrados") — never fabricate a percentage.
- **No "participation score", no ranking, no comparison to other legislators.** Raw counts only.

### 3.4 Voto × tema (facet by materia)

A filter, not a verdict. Reuses v1.0 embeddings/materia.

```
<h3 className="text-sm font-semibold">Por tema</h3>
[ materia filter — chips or shadcn Tabs, server-driven via ?materia=slug ]
[ filtered vote list reuses §3.2 rows ]
[ optional: "Votó N de M veces en proyectos sobre {tema}" — text-sm text-muted-foreground ]
```

- **Filter UI:** prefer server-driven anchor chips (`Badge`-styled `<a href="?materia=...">`,
  `min-h-[44px]`) over client tabs. The active chip uses `border-b-2 border-primary` (same active
  affordance as v1.0 section nav). If shadcn `Tabs` is used, it must remain a thin client island.
- **Descriptive only (VOTE-03, anti-feature LOCKED):** each tema shows *raw tallies* — "Votó X de Y
  veces en proyectos sobre {tema}". **PROHIBITED:** any conviction/ideology/left-right score, any
  "vota alineado con / en línea con", any % framed as a stance, any heat/intensity coloring of a
  tema. The reader infers; the product never editorializes.
- Each filtered row keeps its own ProvenanceBadge and boletín link.

### 3.5 Rebeldías (raw count + list)

```
<h3 className="text-sm font-semibold">Votó distinto a su bancada</h3>
[ "Votó distinto a la mayoría de su bancada N veces" — text-base ]
[ list of those votaciones — reuses §3.2 VotoFichaRow, each with boletín + ProvenanceBadge ]
```

- **Raw count + the underlying list only (VOTE-04, LOCKED).** Neutral label exactly: **"votó
  distinto a su bancada"** — never "rebeldía/rebelde/díscolo/leal/disciplina" in user-facing copy
  (the section heading uses the neutral phrasing; "Rebeldías" is an internal name only). No %, no
  "loyalty index", no judgment.
- **Honest method (must be visible):** the bench majority is computed **from the same votación**
  (the option the majority of the legislator's bancada cast in that division), never inferred.
  Surface this as a one-line footnote: "Se compara el voto del parlamentario con la opción
  mayoritaria de su bancada en esa misma votación." Requires bancada-at-time-of-vote from the
  maestra.
- **Empty:** "No se registran votaciones en que haya votado distinto a su bancada." — this is a
  fact, NOT a virtue. Do not phrase as "100% alineado" or any praise.

### 3.6 The three honest states (LOCKED — VOTE-05, applies per row AND per sub-block)

A vacío NUNCA se lee como "limpio/sin votos". Three distinct states:

| State | Trigger | Rendering |
|-------|---------|-----------|
| **(a) enlazado-confirmado** | `estado_vinculo='confirmado'` AND `parlamentario_id` set | Full row; legislator name/profile is the trusted subject; boletín links to `/proyecto/[boletin]`; ProvenanceBadge present. This is the normal case for Cámara DIPID votes. |
| **(b) presente-no-verificado** | source lists the name but link is `probable`/`no_confirmado` | Show the **raw mention** (`mencion_nombre`) + `<IdentityMarker />`. **NEVER** rendered as a link to this profile and **never aggregated into** attendance/rebeldías counts for this legislator. Lives in a clearly-labeled "menciones sin verificar" area, not in the confirmed list. |
| **(c) no-ingestado** | no rows for this legislator / section (data not yet loaded) | Explicit honest empty (see §6.1). Copy states *not yet ingested*, distinguishing "no consultado todavía" from "consultado, sin votos". Never blank, never reads as a clean record. |

State (a) vs (c) distinction is existential: an empty VOTE section after a successful query that
returned zero confirmed rows must say so honestly — it must not look identical to a legislator with
a full record and no rebellions.

## 6. States

### 6.1 Empty / honest-empty states (LOCKED — never reads as "clean")

| Scope | Trigger | Copy |
|-------|---------|------|
| Parlamentario not found | id not in maestra | "No encontramos a este parlamentario en el registro. Es posible que el identificador sea incorrecto." → `notFound()` |
| VOTE section — no votes ingested | confirmed-vote query returns 0 AND ingestion not yet run for this legislator | "Aún no hemos ingerido las votaciones de este parlamentario. Esto no significa que no haya votado — los datos se están incorporando." |
| VOTE section — ingested, zero confirmed | ingestion ran, 0 confirmed rows | "No hay votaciones confirmadas para este parlamentario en la legislatura vigente." |
| Voto×tema — no votes in tema | filter returns 0 | "No hay votaciones registradas en proyectos sobre {tema}." |
| Rebeldías — none | 0 divergent votes | "No se registran votaciones en que haya votado distinto a su bancada." |
| Asistencia — denominator unknown | membership-at-date uncomputable | Show raw numerator only: "Emitió N votos registrados." (no fabricated %) |

Copy principles (v1.0 §6.1, hardened): state the fact, give context, never say "error" for missing
data, never "Oops", **never let an empty read as a virtue or a clean record.** Distinguish "no
ingestado todavía" from "ingestado, sin resultados".

### 6.2 Loading states

shadcn `Skeleton`, shape-matched, one per Suspense boundary (`ParlamentarioHeaderSkeleton`,
`VotosSkeleton`). Skeleton containers `aria-hidden="true"` (no focus). Mirror the real structure:
header = chip row + title bar + line; votos = 3 grouped-row skeletons + a bar skeleton for
asistencia. Pattern identical to `/proyecto/[boletin]` skeletons.

### 6.3 Error states

Inherited from v1.0 §6.3. Supabase query failure renders inside
`border border-destructive/20 bg-destructive/5 rounded-lg p-4 text-sm`:

"Ocurrió un error al cargar las votaciones. Si el problema persiste, puedes consultar la fuente
directamente en [fuente oficial ↗]." + `<button>Reintentar</button>` (`text-primary underline`).

Distinguish a real DB/network error (throw → honest error UI) from "no rows" (→ §6.1 empty). Use
`.maybeSingle()`/checked errors as in `/proyecto/[boletin]` (`#34` pattern) — never mask a transient
error as "sin votos".

**Provenance unavailable:** render ProvenanceBadge with `sourceName="fuente desconocida"`, no link.
Never hide the badge — its absence falsely implies the datum has no source.

### 9.1 Tone rules — HARD CONTENT RULE: anti-causality / anti-affinity (riesgo existencial #2)

These are enforced at the component and copy level and are the most important constraint of the
phase. A checker/auditor failure on any of these BLOCKS the phase.

1. **No causality, ever.** Never assert or imply *why* a legislator voted. No "por presión de",
   "a cambio de", "favoreciendo a", "respondiendo a".
2. **No affinity / alignment language.** PROHIBITED verbatim: "vota alineado con", "en línea con",
   "afín a", "aliado/rival", "vota junto a", co-voting "most often with". Voto×tema and rebeldías
   are *observable data*, never relationship claims.
3. **No score / index / ranking of any kind.** No ideology score, left-right placement, voting-power
   / decisiveness index, loyalty/discipline index, participation score, or comparison-to-peers.
   Raw tallies only; the reader infers.
4. **No judgment adjectives** about a vote, an absence, or a rebellion ("polémico", "histórico",
   "díscolo", "leal", "rebelde", "controversial").
5. **Identity uncertainty** uses exactly "identidad no verificada" — never "posible", "probable",
   "dudoso", or any hedge that implies a guess.
6. **Provenance is mandatory** on every row; if unknown, "fuente desconocida" — never omitted.
7. **An empty is a fact, not a virtue.** Never phrase absence of data as cleanliness, alignment,
   or praise.
8. **Neutral voice for absences/abstentions** — "Ausente" / "Abstención" are rendered faithfully,
   never collapsed into "no votó" and never editorialized as evasion.

## 10. Provenance & Identity Data Flow (UI contract with backend)

Every vote row carries:

```ts
interface VotoFichaRow {
  votacion_id: string;
  boletin: string;                 // → /proyecto/[boletin]
  fecha: string;                   // ISO; votación date
  seleccion: "si" | "no" | "abstencion" | "pareo" | "ausente";  // enum EXTENDED with "ausente"
  estado_vinculo: "confirmado" | "probable" | "no_confirmado" | null;  // identity guard
  parlamentario_id: string | null;
  mencion_nombre: string;          // raw source name (for state (b))
  origen: string;                  // → sourceLabel() → ProvenanceBadge.sourceName
  fecha_captura: string;           // → ProvenanceBadge.capturedAt
  enlace: string | null;           // → ProvenanceBadge.sourceUrl (source votación record)
}
```
