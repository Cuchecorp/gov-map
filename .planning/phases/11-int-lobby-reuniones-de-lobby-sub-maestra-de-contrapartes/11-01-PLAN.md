   - `estado_vinculo text` — 'confirmado' | 'no_confirmado' | null.
   - `fecha timestamptz`, `fecha_raw text` (string fuente "2023-12-26 13:00:00-03" si el parseo falla — nunca fabricar).

   - `alter table lobby_audiencia enable row level security;` + `create policy lobby_audiencia_public_read on lobby_audiencia for select to anon using (true);` + `grant select on lobby_audiencia to anon;`
   - `create index lobby_audiencia_parlamentario_idx on lobby_audiencia (parlamentario_id);`

2. **`lobby_contraparte`** (SUB-MAESTRA, DENY-BY-DEFAULT — copia VERBATIM la convención de 0018):
   - `id bigint generated always as identity primary key`.
   - `identificador text not null references lobby_audiencia(identificador) on delete cascade`.
   - `nombre text not null` (nombre crudo del tercero), `rol text not null default ''` (rol crudo; '' si ausente), `representado_text text` (firma/entidad representada, raw, nullable).
   - `contraparte_id text` — NULLABLE, uso interno, NO es un FK a parlamentario; default NULL (en P11 esencialmente siempre NULL — no hay cross-key autoritativo de tercero). Comentar que NUNCA se puebla por adivinanza de nombre (Pitfall 4).
   - Provenance inline NOT NULL: `origen`, `fecha_captura`, `enlace`.
   - `unique (identificador, nombre, rol)` — clave natural para upsert idempotente.
   - `alter table lobby_contraparte enable row level security;` — y NINGUNA policy, NINGÚN grant a anon (deny-by-default per 0018). Comentar explícitamente la ausencia intencional, como hace 0018.
   - NO existe columna RUT de contraparte en P11 (no se almacena RUT de terceros — minimización Ley 21.719). Si una columna sensible se agregara aquí en el futuro, la RLS deny-by-default la protege por construcción.

3. **`lobby_de_parlamentario(p_id text)`** (RPC público — espejo EXACTO de `parlamentario_publico`/`votos_de_parlamentario`):
   - `language sql stable security definer set search_path = ''`.
   - `returns table (identificador text, fecha timestamptz, fecha_raw text, materia text, enlace_detalle text, origen text, fecha_captura timestamptz, enlace text, contraparte_nombre text, contraparte_rol text, representado text)`.
   - Cuerpo: `select a.identificador, a.fecha, a.fecha_raw, a.materia, a.enlace_detalle, a.origen, a.fecha_captura, a.enlace, c.nombre, c.rol, c.representado_text from public.lobby_audiencia a left join public.lobby_contraparte c on c.identificador = a.identificador where a.parlamentario_id = p_id order by a.fecha desc nulls last;`
   - SOLO devuelve audiencias confirmadas (FK = p_id) y SOLO campos que la fuente ya publica — NUNCA `contraparte_id`, NUNCA un RUT, NUNCA `institucion_codigo` interno-irrelevante si no aplica. Esto materializa el estado-honesto: una audiencia no-confirmada nunca aparece bajo un parlamentario.
   - `revoke execute on function public.lobby_de_parlamentario(text) from public;` + `grant execute on function public.lobby_de_parlamentario(text) to anon;`

4. **`lobby_ingesta_estado`** (marcador de ingesta por-parlamentario — resuelve Open Question 3, el gap real de `VotosView.noIngestado` hardcodeado a false):
   - Tabla mínima: `parlamentario_id text primary key references parlamentario(id) on delete cascade`, `ingestado_hasta date`, `fecha_captura timestamptz not null default now()`.
   - public-read (la ficha la consulta con anon para distinguir estado (a) "no ingestado" de (b) "ingestado, cero confirmadas"): `enable row level security` + `create policy ... for select to anon using (true)` + `grant select to anon`. NO contiene PII (solo un FK + fecha) → public-read es correcto.
   - Comentar: una fila presente = "ya corrió la ingesta de lobby para este parlamentario"; su ausencia = "no ingestado todavía". El conector (Plan 02) la upserta al final de una corrida que tocó a ese parlamentario.
