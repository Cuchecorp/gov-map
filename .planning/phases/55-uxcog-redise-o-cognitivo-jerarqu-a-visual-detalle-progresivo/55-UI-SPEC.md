## Spacing Scale

Declared values (all multiples of 4; consumed as Tailwind spacing tokens, no arbitrary color values):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, chip inner padding, meta-line gaps |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, rail nav item padding |
| md | 16px | `gap-4` / `p-4` | Default element spacing, row padding, card padding-min |
| lg | 24px | `gap-6` / `p-6` | Section (`b-sec`) internal padding; gap between sections in the rail layout |
| xl | 32px | `gap-8` | Layout gap between rail column and content column |
| 2xl | 48px | `mt-12` | **LOCKED carril frontier** — see Exceptions |
| 3xl | 64px | `py-16` | Page-level vertical padding (existing `py-8 md:py-16`) |

Exceptions (all multiples of 4 — no arbitrary values):
- **`mt-12` (48px) is the LOCKED anti-insinuación carril frontier** (DESIGN-SYSTEM
  §3/§8). Every domain carril (`<section>`) remains a sibling separated by `mt-12`.
  The sketch's tighter 24px between `.b-sec` cards is **overridden**: keep `mt-12`
  between domain carriles. Internal card padding is 24px (`p-6`); frontier between
  carriles stays 48px. NEVER move `mt-12` onto a wrapper, and never nest two domains.
- **Touch targets: `min-h-11` (44px)** on every interactive control (rail nav links,
  "Ver detalle" buttons, disclosure triggers). Existing pattern — preserve.
- **Rail column width: 208px (`w-52` / grid track `13rem`)** — the sketch's ~210px
  rounds to the token 208px (13rem, a spacing multiple). Do NOT use `w-[210px]`.
- **Sticky offsets: `top-16` (64px) + `scroll-mt-16` (64px)** — rail sticky offset and
  section scroll-margin below the global header. Multiples of 4.
- Arbitrary-value ban clarification (gotcha 54-04): the ban targets **color tokens**
  (`@theme inline` without double-`hsl()`). A layout **grid track** such as
  `md:grid-cols-[13rem_1fr]` is geometry, not a color, and is permitted where no named
  utility exists.

## Typography

Exactly 4 sizes, exactly 2 weights (400 regular / 600 semibold). The sketch's `700`
KPI weight is **downgraded to 600** to respect the LOCKED 2-weight system; preattentive
emphasis comes from SIZE + Mono + semantic COLOR, not from a third weight.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Meta / label | 12px (`text-xs`) | 400 | 1.4 | Mono for dates/counts, Sans for labels | Row meta line (`fecha · fuente ↗`), rail counts, provenance badge, section count suffix |
| Body | 14px (`text-sm`) | 400 | 1.5 | Sans | Row primary line, rail nav labels, intro/caveat copy |
| Heading | 18px (`text-lg`) | 600 | 1.2 | Sans | Section `<h2>`, rail name `<h1>` |
| Display (KPI figure) | 24px (`text-2xl`) | 600 | 1.1 | **Mono** | Capa-1 preattentive numbers (72 / 66 / 99,3% …) — the scannable layer |

Rules:
- ALL figures, counts, dates, percentages, week codes → **Geist Mono** (existing convention).
- Body copy line-height 1.5; headings and KPI figures 1.1–1.2 (tight, for scanning).
- No `font-bold`/`700` anywhere. Emphasis = size + Mono + semantic color only.
- Capa-1 row text is compressed to **1 primary line + 1 meta line** (`text-xs` muted
  Mono). NO raw source paragraphs in capa-1 (they move into the expanded detail).

  (`text-lg`/600), (c) período (Mono `text-xs` muted), (d) compact frescura/provenance

  (`Votaciones 141`, `Lobby 107`, `Patrimonio 10`, `Cruces ◆ 12`, `Financiamiento —`),

- **Content:** each carril = its own `<section id className="mt-12">` sibling (frontier

**Mobile (< md):** rail collapses to a compact top bar — horizontally scrollable local

Cruces (gated) → Financiamiento (real gated OR pendiente). Only carriles actually in the
HTML appear in the rail (respect crucesPublicEnabled / moneyPublicEnabled gates exactly).

## Progressive-Disclosure & Interaction Contract

Three cognitive layers (Shneiderman: overview → details-on-demand):

| Layer | What | Mechanism |
|-------|------|-----------|
| Capa 1 (overview) | Preattentive summary: big Mono figures + 1 visual + count. Always visible. ≤7 chunked units per collapsed section. | Server-rendered, always in DOM |
| Capa 2 (detail) | Truncated row list (5–10 rows) + "Ver las N" | Radix Collapsible/Accordion, client-side toggle over ALREADY-FETCHED data (NO lazy-fetch, NO new RPC) |
| Capa 3 (traceability) | fuente · fecha · enlace on every row's meta line + section frescura badge | Always present, never competes for attention (Mono, muted, `text-xs`) |

Rules:
- **Server Components by default.** All ficha data already arrives complete at server
  render. Disclosure is a CLIENT toggle over data already in the HTML (Radix
  `forceMount` keeps SSR content in the DOM, F45 pattern) — it is NOT a lazy-fetch and
  introduces NO new RPC/query.
- **Truncation:** initial detail shows ~5–10 rows; "Ver las N" reveals the rest via
  client-side incremental reveal over the already-present rows (expand-all or paginate —
  planner's discretion). No round-trip.
- **Scrollspy:** IntersectionObserver only (zero deps); marks the rail current item.
- **Default open state:** capa-1 of every section is always visible; the row DETAIL is
  collapsed by default. (This replaces the F45 heuristic of auto-opening data carriles —
  in the rail pattern the summary IS the always-visible layer, so details start closed
  to hit the ~5.000px default target.)
- All disclosure controls are `min-h-11` and keyboard-operable (Radix handles ARIA).

## Per-Surface Contract

### `/parlamentario/[id]` — full rail pattern (primary deliverable)
- Rail + content per Layout Contract. Target default height ≈ **5.000px logical**
  (from 28.048px), zero data lost.
- Capa-1 per carril:
  - **Votaciones:** 5 Mono facts (a favor / en contra / abstención / ausente /
    asistencia %) + existing "Cómo votó" stacked bar. Detail = existing `VotosSection`
    (truncated).
  - **Lobby:** top-N materia horizontal bars (CSS on `--accent`? NO — use neutral
    `--muted-foreground`/border bars; petróleo is reserved) + count. Detail = existing
    `LobbySection` (truncated), contraparte name plain non-linked (LOCKED 52-03).
  - **Patrimonio:** compact per-declaration mini-column strip (counts only, NEVER
    montos — F46 rule) + "10 declaraciones · 2019–2026". Detail = existing
    `PatrimonioSection` (Recharts chart F46 lives here, in the detail).
  - **Cruces (gated):** petróleo-framed. Sector chips "sector · N reuniones · M votos"
    + primary "Explorar los N cruces". Detail = existing `CrucesSection`.
  - **Financiamiento:** real gated sections OR the `opacity-60` pendiente section with
    the LOCKED legal line; present in rail as "—".

### `/proyecto/[boletin]` — rail + stepper
- Rail sections: Dónde está / Tramitación / Votaciones / Lobby del período / Idea matriz / Similares.
- **Tramitación capa-1 = visual etapa stepper** (elevate the existing "¿Dónde está hoy?")
  + KEY hitos always visible: ingreso, cambios de etapa/comisión, informes de comisión,
  votaciones con desenlace, urgencia vigente. Repetitive urgencia trámites AGRUPADOS into
  one line with count ("42 trámites de urgencia · ver todos"). Grouping is presentational
  over data already classified by tipo/urgencia; when the heuristic is ambiguous, group
  ONLY the unambiguous rows. Full tramitación timeline = collapsed detail.
- **Lobby×tramitación carril:** capa-1 = count + week; detail inline; contraparte name
  plain non-linked (LOCKED 52-03).

### `/agenda` — group día → comisión
- Group by day, then by comisión, with clear typographic hierarchy. Collapsible blocks
  per day. Cross-links to boletín intact. (From 11.606px to a scannable grouped view.)

### `/red` — ego-network initial view
- With `?seed=`: center + zoom to the seed node's immediate neighborhood (`fitView` over
  filtered neighborhood `nodes`, `minZoom`), NOT a global fitView of 136 nodes.
- Without seed: keep global fitView + the usage note. Use only already-available
  `@xyflow/react` props (fitView on filtered nodes, `nodes`, `minZoom`) — zero new
  physics, zero new dep.

### NOT touched
- Home and `/buscar` (F54 left them good). Hero LOCKED.
