<objective>
Cierra el walking skeleton: orquestacion por cola que dirige el framework de ingesta. Migracion 0003 crea las queues pgmq (`ingest_jobs`, `ingest_dlq`), la funcion dispatcher `util.process_ingest_jobs()` y el `cron.schedule` que la dispara; la Edge Function `ingest-worker` (Deno) desencola un lote, corre el `DummyConnector` (heredando `BaseConnector` de `@obs/ingest` via import map) y hace ack en exito. Fallos reaparecen via visibility timeout (backoff ante 429); venenosos van a DLQ. Un workflow de GitHub Actions queda como escape hatch para backfill masivo que excede ~400s.

Purpose: FND-05 — la ingesta pesada corre dirigida por cola con chunking y backoff, no como un run largo que revienta el limite de ~400s de Edge Functions. Es el ultimo eslabon que hace recuperable y re-procesable la rebanada de ingesta end-to-end.
Output: orquestacion versionada en SQL (no en el dashboard), worker que ejercita el DummyConnector end-to-end via la cola, escape hatch CI, y un checkpoint humano que verifica el bucket R2 real + el PUT condicional.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pg_cron/pg_net ↔ Edge Function | el dispatcher invoca el worker con service_key; el secret no debe estar en la migracion en claro |
| cola ↔ worker | el worker no debe fetchear hosts arbitrarios de la cola (SSRF) |
| Edge runtime ↔ secrets | R2_*/SUPABASE_* via supabase secrets, nunca hardcode ni cliente |
| poison message ↔ cola | un mensaje veneno no debe loopear infinitamente (DoS interno) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Information Disclosure | service_role key hardcodeada en migracion 0003 | mitigate | Usar `util.service_key()`/vault, no literal; verify greppea ausencia de la key en claro |
| T-01-10 | Elevation of Privilege | service key filtrada al worker/log o al cliente | mitigate | Secret solo en Edge Function via supabase secrets + repo secrets en CI; nunca en respuesta ni frontend |
| T-01-11 | Denial of Service | poison message satura ingest_jobs | mitigate | read_ct > umbral → pgmq.archive a ingest_dlq; el worker no entra en loop infinito (FND-05 DLQ) |
| T-01-12 | Tampering | SSRF: worker fetchea URL arbitraria del mensaje de cola | mitigate | El BaseConnector valida host contra allow-list; en M1 solo DummyConnector con host declarado |
| T-01-13 | Denial of Service | run unico largo topa el limite ~400s de Edge Functions | mitigate | Chunking obligatorio (lote pequeno por invocacion) + escape hatch GitHub Actions para backfill masivo |
| T-01-SC | Tampering | npm/jsr installs en deno.json (supply chain) | accept | aws4fetch/robots-parser auditados [OK] en npm (RESEARCH); supabase-js/zod oficiales. Sin paquetes [ASSUMED]/[SUS] nuevos en esta plan |
