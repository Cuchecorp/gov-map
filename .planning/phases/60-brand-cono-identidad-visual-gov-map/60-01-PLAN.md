# Phase 60 Plan 01: Brand Assets & Header Lockup

## Objective

Integrate the selected "C — Capas que se cruzan" icon (LOCKED geometry from 60-SELECTION.md) into the gov-map app: generate all favicon/icon/OG assets from the master SVG, wire the icon into the GlobalHeader lockup, and update layout metadata. Deploy deferred to Phase 61.

## Tasks

### Task 1 — Brand Assets

**type:** auto

Generate and commit all brand static assets:

1. `app/app/icon.svg` — master SVG, Next.js file convention.
2. `scripts/generate-brand-assets.mjs` — Node script using sharp (devDep) to produce:
   - `app/app/favicon.ico` (16/32/48 multi-res)
   - `app/app/apple-icon.png` (180×180, cream bg #FAF8F3)
   - `app/app/opengraph-image.png` (1200×630, cream bg, icon ~200px + text)
   - `app/public/icon-192.png` and `app/public/icon-512.png` for manifest
3. `app/app/manifest.ts` — Next.js metadata manifest export with name, icons, theme/bg colors.

Fallback: if sharp cannot produce .ico on Windows, use png-to-ico or encode manually; if all fail, ship SVG-only and document.

**done:** All asset files committed; manifest.ts type-checks; no build errors.

### Task 2 — Header Lockup & Tests

**type:** auto

1. Create `app/components/brand-icon.tsx` — inline SVG React component (currentColor support, size prop).
2. Update `app/components/global-header.tsx` — replace text-only brand with `[BrandIcon 24px] gov-map` lockup. Preserve all nav, typography, spacing.
3. Update layout metadata in `app/app/layout.tsx` — ensure og:image points to the file-convention path; update title to "gov-map" if appropriate per product direction.
4. Add/update header structural test asserting lockup contains SVG + "gov-map" text.

**done:** `pnpm --filter app test --run` green; `pnpm -w typecheck` clean; `pnpm --filter app build` green.
