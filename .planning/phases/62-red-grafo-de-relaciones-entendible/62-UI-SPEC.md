# Phase 62 — UI Design Contract

> Visual and interaction contract for `/red` — el grafo de relaciones entendible (RED-01/02/03).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>

Every value is pre-populated from CONTEXT.md (locked decisions), REQUIREMENTS.md (RED-01/02/03), or the existing design system in `app/app/globals.css` + `app/app/styles/civic-tokens.css` + `app/components/red/*`. Where CONTEXT.md left discretion, decisions follow the shipped `/red` conventions and are noted as `[researcher]`.

## Scope Recap (what this contract governs)

Only the `/red` surface: `app/app/red/page.tsx` + `app/components/red/{red-graph,nodo-parlamentario,arista-hecho}.tsx` + the `.net-*` block in `app/app/globals.css`. **Zero DDL**, zero flags, zero materializer changes. The failure this contract fixes: with a seed the graph today renders ~136 crammed nodes in a horizontal band (the neighborhood filter does not limit what is rendered; the deterministic-grid layout is illegible at that scale). The target: a real ego-network (seed + direct neighbors only) in a deterministic **radial/orbital** layout with labels legible without zoom, an honest neighbor cap, and a móvil-390px fallback.

**LOCKED invariants (F18 / anti-insinuación — never violate):**
- Zero force-simulation. Position never encodes affinity or closeness between people.
- Node = public identity only (nombre + cámara). Never partido, never face image, never a scoring/ranking badge.
- Edge = a sourced, typed, time-windowed fact. Copy describes the fact, never a valuation, proximity, or motive.
- Empty graph (0 aristas) = honest state, never an error.
- Provenance (fuente + ventana + enlace + licencia) always in the DOM, not hover-only.

## Spacing Scale

Declared values (Tailwind default 4px base — multiples of 4 only; the shipped `.net-*` block already uses these):

| Token | Value | Usage in `/red` |
|-------|-------|-----------------|
| xs | 4px | Icon gap in edge label, node line gap (`gap-0.5`) |
| sm | 8px | Compact spacing: legend list items, filter checkbox gap |
| md | 16px | Default: page horizontal padding móvil (`px-4`), legend padding, canvas top margin (`mt-4`) |
| lg | 24px | Section gaps between legend / filters / canvas (`mt-6`) |

- Radial layout geometry (radius, ring spacing) is **pixel geometry inside the SVG canvas**, not page spacing — it is NOT bound to the 4px scale. See "Radial Layout Contract" below for its own tokens.

60/30/10 split, inherited from Phase 19 (LOCKED, `app/app/globals.css`). Values are HSL triplets consumed as `hsl(var(--token))`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `40 33% 97%` (crema) | Page background |
| Secondary (30%) | `--card` `40 30% 99%` + `--muted` `40 20% 93%` | Canvas background (`.net-lienzo` = card), node background (card), legend/filters panel (muted), border `--border` `40 16% 86%` |
| Accent (10%) | `--accent-product` `183 38% 26%` (petróleo) | **Edge stroke** (`.net-arista__path`), **provenance links** ("Ver fuente oficial"), **checkbox accent-color**, focus ring (`--ring`). Nothing else. |
| Destructive | `--destructive` `0 72% 42%` | **Not used this phase** — `/red` has zero destructive actions. |

**Accent reserved for (explicit — never "all interactive elements"):**
1. Edge lines (the fact connectors).
2. Provenance/source hyperlinks.
3. Checkbox/control accent-color and the focus-visible ring.

**Cámara distinction (RED-02 — "cámara distinguible por forma/borde del nodo, no por posición"):**
- Distinguish Cámara vs Senado by **node border treatment**, NOT by radial position and NOT by political color.
- Use the civic institutional tokens already defined in `civic-tokens.css`: `--camara` (institutional blue `213 94% 38%`) and `--senado` (institutional burgundy `355 65% 38%`) as a **left border accent or a small shape/outline cue on the node** — these are institutional identity colors, explicitly NOT party colors (documented in `civic-tokens.css` header). `[researcher]` Recommended: 3px left border in the cámara-muted tone, keeping the node body crema. Keep contrast subtle so it reads as institutional wayfinding, not as valuation.
- The **seed node** keeps its existing sober marker: `.net-nodo--seed` = 2px border in `--foreground` (neutral ink), **never petróleo, never a score badge**. Petróleo is reserved for edges/links only.

**Anti-insinuación color rule (LOCKED):** the accent (petróleo) is NEVER applied to a node fill or a node border — a colored node would read as a ranked/highlighted person. Nodes stay crema/card with institutional-neutral borders only.

## Radial Layout Contract (RED-02 — the core visual change)

Replaces the current `posicion(laneIndex, camara)` grid (`red-graph.tsx` ~L112) with an **ego-centric radial/orbital deterministic layout**. This is the heart of the phase.

**Geometry (SVG-canvas pixels, not page spacing):**

| Token | Value | Rationale |
|-------|-------|-----------|
| Seed position | canvas center `(0, 0)` | Ego at center |
| Ring 1 radius | ~260px | First ring of direct neighbors; keeps 160px-wide nodes from overlapping up to ~10–12 neighbors |
| Ring 2 radius | ~460px | Overflow ring when neighbors exceed ring-1 capacity `[researcher]` |
| Angular order | **alphabetical by neighbor name, clockwise starting at 12 o'clock** | Neutral, declared order — NEVER by edge weight/count |
| Node min-width | 160px (`.net-nodo`, unchanged) | Label legibility without zoom |
| Neighbor cap (hard) | **24 rendered neighbors** `[researcher]` (CONTEXT suggested p.ej. 24) | Legibility ceiling; above this, truncate honestly |

**Determinism (F18 LOCKED):** position is a pure function of `(alphabetical index, neighbor count)`. No physics, no randomness, no weight input. Given the same neighbor set, the layout is byte-identical every render. The legend MUST state this explicitly (see Copywriting).

**Neighbor ordering rule (LOCKED, declared):** neighbors are placed **alphabetically by display name**. The ordering criterion is neutral and stated in the legend. Never order by "peso", co-occurrence count, or any measure that implies importance.

**Cap + honest truncation (RED-01):**
- Render at most 24 neighbors on the ring(s).
- If the seed has > 24 direct neighbors, show the first 24 (alphabetical) and render an honest, non-hover control: **"N vecinos más"** that either expands the ring or reveals a plain text list of the remaining neighbor names (each a link to `/red?seed=<id>` for that neighbor). The count N must be truthful (total direct neighbors − rendered). Never silently drop neighbors.
- Visible node count in the DOM MUST be **≤ neighbors_rendered + 1 (the seed)** — verifiable by the checker/executor by counting `.net-nodo` elements. This is the RED-01 acceptance assertion.

**Móvil 390px (RED-02 — "usable en móvil"):** `[researcher]` decision — **degrade to a vecinos list, not a shrunken ring**, below the `md` breakpoint (768px). At 390px a 24-node ring is illegible; the honest, more usable form is a heading + a linked list of direct neighbors, each row showing the neighbor name, cámara, and the shared-fact summary + provenance (reusing `arista-hecho` copy). The seed name stays prominent at top. Keep the desktop radial for ≥768px. The existing móvil note ("El grafo se lee mejor en pantalla ancha…") is replaced by this real list — the note was a band-aid; the list is the fix.

## Node & Edge Contracts (reuse + adjust)

**Node (`nodo-parlamentario.tsx`) — keep as-is, add cámara border cue:**
- Projects ONLY `nombre` (via `formatNombre`) + cámara label. No partido, no face, no RUT, no badge. (Already enforced.)
- Add the institutional cámara border cue (see Color). Seed keeps `--net-nodo--seed` neutral 2px marker.
- `aria-label`: `Parlamentario: {nombre}` (+ ` (punto de partida)` for seed). Unchanged.

**Edge (`arista-hecho.tsx`) — keep fact copy + provenance; handle density:**
- Label describes the fact via `etiquetaHecho` (e.g. "Ambos recibieron audiencia de {contraparte}") + `ventanaTexto`. Provenance dl always in DOM + Radix tooltip mirror. Petróleo stroke. (All unchanged.)
- **Density handling (CONTEXT):** if seed↔neighbor pairs produce many parallel edges (a pair sharing N facts), **aggregate per pair**: one line per pair whose label reads "N hechos en común" with a "ver los N registros" expander that lists each fact + provenance. One visible line per pair keeps the ring readable. `[researcher]` Only aggregate when a pair has > 1 fact; single-fact pairs render their fact inline as today.

## Page States (all must be honest)

| State | Trigger | What renders |
|-------|---------|--------------|
| **No seed** | `/red` no `?seed` | Heading + orienting explanation + prominent JS-free `<select>` of parlamentarios (grouped Cámara/Senado) + "Ver relaciones" button. **Never the full graph.** (Shipped in `page.tsx` L69-131 — keep; ensure the selector reads as the prominent primary action.) |
| **Seed, has neighbors** | valid seed, ≥1 arista | Heading + intro + "Centrado en {nombre}…" subline + radial ego-network (desktop) / vecinos list (móvil) + rewritten legend + filters. |
| **Seed, empty graph** | valid seed, 0 aristas | Honest empty copy ("Aún no hay relaciones…") + link back to `/parlamentarios`. NEVER an error. (Shipped — keep.) |
| **Filters exclude all** | filters hide every edge | "Ninguna relación coincide con los filtros…" (Shipped — keep.) |
| **DB/RPC failure** | `subgrafo_red` or `parlamentarios_publico` errors | Throw → error boundary. NEVER degrade to "sin relaciones" (a real failure must not read as a clean empty). (Shipped — keep.) |
| **Truncated** | > 24 direct neighbors | 24 rendered + honest "N vecinos más" expander/list. |

## Copywriting Contract

All copy in Spanish, sober, anti-insinuación. Never causal/affinity/score language.

| Element | Copy |
|---------|------|
| Page heading | `Relaciones entre parlamentarios` |
| No-seed intro | `Elige un parlamentario para ver con qué otros comparte hechos públicos —por ejemplo, haber recibido audiencia de la misma contraparte de lobby—. Cada relación se muestra con su fuente y su fecha.` |
| Primary CTA (no-seed) | Button: `Ver relaciones` · select placeholder: `Elige un parlamentario…` |
| Seed intro | `Hechos públicos que vinculan a este parlamentario con otros. Cada relación es un hecho con fuente y fecha; no afirma intención ni causa.` |
| Seed subline | `Centrado en {nombre} y su vecindario inmediato.` (fallback sin nombre: `Centrado en el parlamentario seleccionado y su vecindario inmediato.`) |
| Legend heading | `Cómo leer este grafo` |
| Legend — nodo | `Nodo: un parlamentario con identidad confirmada (nombre y cámara). El punto de partida se muestra al centro con un borde reforzado.` |
| Legend — línea | `Línea: un hecho público que vincula a dos parlamentarios, por ejemplo haber recibido audiencia de la misma contraparte de lobby. Cada línea lleva su fuente, ventana temporal y enlace al registro original.` |
| Legend — layout (REWRITTEN for radial, RED-02) | `La posición en el anillo es orden alfabético, no cercanía: el parlamentario elegido va al centro y sus vecinos se ordenan alfabéticamente alrededor. La distancia o el ángulo no indican afinidad ni relación entre las personas.` |
| Legend — source line | `Fuente: Ley del Lobby (Ley 20.730) · datos ingestados por este observatorio.` |
| Neighbor cap / truncation | `Se muestran los primeros 24 vecinos en orden alfabético. Ver {N} vecinos más` (expander/list) |
| Aggregated edge | `{N} hechos en común` · expander: `Ver los {N} registros` |
| Empty state (seed, 0 edges) | `Aún no hay relaciones para mostrar para este parlamentario. Cuando existan hechos públicos que vinculen a dos parlamentarios —por ejemplo, haber recibido audiencia de la misma contraparte de lobby— aparecerán aquí, cada uno con su fuente y su fecha.` + link `directorio de parlamentarios →` |
| Filters-exclude-all | `Ninguna relación coincide con los filtros seleccionados. Ajusta el tipo o el periodo para ver los hechos disponibles.` |
| Móvil vecinos list heading `[researcher]` | `Vecinos de {nombre}` |
| Error state | Handled by throw → Next error boundary (no inline copy). A real failure MUST NOT render as an empty graph. |
| Destructive confirmation | None — `/red` has zero destructive actions. |

**Banned vocabulary (LOCKED — must stay green):** cercano/cercanía as affinity, afinidad, aliado, influencia (as a claim about a person), red de poder, conexión (as causal), score, ranking, "más importante", "vinculado a" in a possessive/causal sense. Provenance and factual framing only.
