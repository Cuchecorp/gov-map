## Layout & States

Single-column block inside `<main className="max-w-3xl mx-auto">` (inherited shell). Mirror `lobby-de-parlamentario.tsx` structure: an honest intro line, then one of three mutually exclusive states.

### Intro (always rendered above any state)
Two muted `text-sm` lines:
1. Framing: contracts are shown strictly as *associated with the RUT*, never attributed as the parlamentario's own.
2. Source attribution: ChileCompra "mención de la fuente" (see Copywriting). This is the analog of `patrimonio`'s `AtribucionCcBy` but is **NOT** CC BY 4.0.

### The three honest states (visually & textually distinct — LOCKED)

The state is derived server-side from (a) the per-parlamentario ingesta marker (`contratos_ingesta_estado`, mirroring `lobby_ingesta_estado` / `probidad_ingesta_estado`) and (b) whether the internal RUT is populated (IDENT-10 debt) and (c) row count. Mirrors the `noIngestado` boolean pattern but with THREE outcomes, not two.

| State | Trigger (server-derived) | Render |
|-------|--------------------------|--------|
| **No consultado todavía** | No internal RUT for this parlamentario (IDENT-10 debt) OR no row in `contratos_ingesta_estado`. This is the DEFAULT and currently most-common state. | Muted `text-sm` message stating we have not yet queried ChileCompra for this RUT, and that absence of a query is NOT absence of contracts. NEVER reads as "clean". |
| **Consultado sin contratos** | Ingesta marker present AND zero RUT-exact contracts returned. | Muted `text-sm` message stating ChileCompra was queried by RUT on `{fecha de corte}` and returned no contracts associated with this RUT, *as of that date*. Distinct copy from the state above. |
| **Enlazado** | Ingesta marker present AND ≥1 RUT-exact contract. | Neutral count line (only permitted aggregate) + paginated list/table of contracts. |

These three are NOT interchangeable. Copy must make "we did not look" and "we looked and found none" unmistakably different (anti "vacío = limpio" rule).

### Enlazado list — per contract row
Each contract is its own row (`<li>` or `<TableRow>`), never composed with a vote/declaration/meeting (carril rule). Per row:
- **Subject = the proveedor entity**, rendered as `text-base` (the prominent element of the row). See persona-jurídica treatment below.
- Literal contract fields the source publishes (e.g. organismo comprador, monto literal, fecha) as NOUN-label + verbatim value, `text-sm` muted labels — mirror `patrimonio`'s `<dl>` discipline. The UI computes nothing (no summed totals, no "%").
- `ProvenanceBadge` per row (`capturedAt` = fecha de captura, `sourceName` = "ChileCompra", `sourceUrl` = enlace to the official record), pushed right with `ml-auto`.
- Fecha de corte shown per row (the as-of date of the ChileCompra query), distinct from `capturedAt`.

Pagination: server-driven `?contratosPage=N`, `PAGE_SIZE = 20`, identical `<nav>` + "Página N de M" (`font-mono`) markup as siblings. Order/pagination of the list is Claude's discretion (CONTEXT) — default fecha DESC.

### Persona jurídica row treatment (LOCKED — the load-bearing rule)
A contract awarded to a **persona jurídica** (the common case for state contracts) must show the **proveedor entity as the grammatical and visual subject**, structurally separated from any link to the parlamentario:
- Render the proveedor name as the row subject (`text-base`), with its RUT-derived label as `persona jurídica` (muted, parenthetical) — the same crude-text discipline as `ContraparteCruda` in lobby.
- The relationship to the parlamentario is expressed ONLY as the RUT-exact link fact, rendered as a separate muted line such as "Enlazado por RUT al parlamentario" — never collapsed into a possessive ("su contrato", "contrato del diputado").
- NEVER render the parlamentario as the contract subject. NEVER a single sentence that fuses entity + parlamentario into a personal attribution.
- A persona-natural contract (RUT-exact to the parlamentario's own RUT) still uses the same neutral subject framing; even then the heading wording stays "asociados al RUT".

### Loading state
`ContratosSkeleton` in `page.tsx`: `aria-hidden` block of an intro `Skeleton h-4 w-3/4`, an attribution `Skeleton h-3 w-1/2`, and 3 row skeletons (`h-12 w-full rounded-lg`), shape-matched to `ContratosView`. Mirror `LobbySkeleton`.

### Error state
A real DB/RPC/network error is NOT an empty state. The Server Component THROWS on `rpcError` (mirroring siblings' `#34` rule) so the route's error boundary renders an honest error — never silently degrades to "consultado sin contratos". This distinction is mandatory.

## Exposure Gate (LOCKED)

| Condition | Behavior |
|-----------|----------|
| `moneyPublicEnabled(process.env) === false` (DEFAULT) | The entire `<section id="dinero">` — including its `<h2>` — is **not rendered**. The gate is checked in `page.tsx` (or inside `ContratosSection`, returning `null`) BEFORE any Supabase read. No skeleton, no heading, no empty state, no DOM node ships to the client. The section is absent, not hidden-with-CSS. Server-only (`moneyPublicEnabled` imports `server-only`; never read `MONEY_PUBLIC_ENABLED` raw — always via the function, the single chokepoint, WR-02). |
| `moneyPublicEnabled(process.env) === true` (post legal sign-off, F13) | `<section id="dinero">` renders with `<h2>Contratos del Estado asociados al RUT</h2>`, Suspense + `ContratosSkeleton`, then `ContratosSection` resolving to one of the three honest states above. Note: the MONEY tables remain RLS deny-by-default to `anon`; reads go through a security-definer RPC `contratos_de_parlamentario` (mirroring `*_de_parlamentario` siblings). |

The gate is a presentation chokepoint only; it does not change any of the visual contract above — it decides presence/absence of the whole lane.

## Copywriting Contract

All copy in Spanish (es-CL), sober tone, zero causal/affinity/judgment language (anti "máquina de sospechas", Phase 11). NOUN labels, never prose verdicts.

| Element | Copy |
|---------|------|
| Section `<h2>` (EXACT, hard rule) | `Contratos del Estado asociados al RUT` |
| Forbidden headings | `Contratos del parlamentario`, `Contratos del diputado/senador`, `Contratos de {nombre}`, any possessive. The wording is a contract-of-honesty term. |
| Intro line 1 | `Contratos del Estado registrados en ChileCompra (Mercado Público) asociados al RUT de este parlamentario. Se muestran tal como los publica la fuente; la asociación es por RUT exacto y no implica que el contrato sea del parlamentario.` |
| Intro line 2 (source attribution — NOT CC BY 4.0) | `Fuente: ChileCompra — Mercado Público (mención de la fuente). Datos consultados por RUT, con su fecha de corte.` |
| Neutral count (enlazado) | `{n} contrato registrado` / `{n} contratos registrados.` (only permitted aggregate; no sum of montos, no ranking) |
| State — No consultado todavía | `Aún no hemos consultado ChileCompra para el RUT de este parlamentario. Esto no significa que no existan contratos asociados — la consulta por RUT se está incorporando.` |
| State — Consultado sin contratos | `Consultamos ChileCompra por el RUT de este parlamentario (corte al {fecha}) y no se registran contratos asociados a ese RUT a esa fecha.` |
| Empty state must NEVER read as | `Sin contratos ✓`, `Limpio`, `No tiene contratos`, or anything implying a clean/positive verdict. Absence of query ≠ absence of contracts. |
| Persona jurídica subject prefix | `Proveedor: {nombre}` + muted `(persona jurídica)` |
| Persona jurídica link line | `Enlazado por RUT al parlamentario.` (separate muted line; never fused into a possessive) |
| Error state | Honest error via route boundary (thrown). User-facing: `No se pudieron cargar los contratos en este momento.` Never silently shown as "sin contratos". |
| Primary CTA | None. This section is read-only display with zero CTAs and zero mutations. |
| Destructive confirmation | Not applicable — no destructive actions in this phase. |
| Pagination links | `Anteriores` / `Siguientes`, `Página {n} de {m}` (`font-mono`), identical to siblings. |

`sourceLabel()` note (for planner/executor): `app/lib/types.ts` `sourceLabel()` currently has no ChileCompra branch — add one (`o.includes("chilecompra") || o.includes("mercado")` → `"ChileCompra"`) so `ProvenanceBadge.sourceName` reads "ChileCompra", not "fuente desconocida".
