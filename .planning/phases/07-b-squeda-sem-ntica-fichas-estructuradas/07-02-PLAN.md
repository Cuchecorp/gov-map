| Boundary | Description |

| pipeline → fuente gov (texto íntegro) | URL no confiable; debe pasar allowlist/robots antes de fetch (SSRF) |
| pipeline → R2 / Supabase | escrituras con service key server-side; key nunca en logs/errores |
| CI workflow → secrets | API keys vía secrets, nunca cleartext en el YAML |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering (SSRF) | texto-fuente.ts (fetch de link arbitrario) | mitigate | assertAllowedUrl (allowlist gov) + robots ANTES de cualquier fetch; nunca fetch de campo link ciego |
| T-07-06 | Information Disclosure | writer/CLI (service key en logs) | mitigate | service key solo en createClient; errores propagan solo error.message de PostgREST; nunca interpolar la key |
| T-07-07 | Information Disclosure | fichas-backfill.yml (secrets en cleartext) | mitigate | todas las keys vía ${{ secrets.* }}; permissions contents:read; gate de verificación grep contra cleartext |
| T-07-08 | DoS (denial of wallet) | pipeline (cuota LLM/embedding) | accept | pipeline es operator-triggered/cron-gated, no user-triggered; --limite acota el lote; repair-loop clamped en el provider |
