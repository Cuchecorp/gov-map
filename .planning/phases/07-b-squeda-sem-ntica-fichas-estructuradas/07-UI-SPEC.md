## 3. SearchBox

**Purpose:** Capture a natural-language query and navigate to `/buscar?q={query}`. The actual embedding + kNN runs server-side on `/buscar` (CONTEXT.md: "Todo el search es server-only"). `SearchBox` itself does NOT call Gemini or pgvector — it only routes.

**Client Component** (`"use client"`) — it owns a controlled input and a submit handler that pushes the route. This is the only client island the phase introduces.

**Visual:**

```
┌─────────────────────────────────────────────┐  [ Buscar ]
│ 🔍  Busca por idea o número de boletín…       │
└─────────────────────────────────────────────┘
```

**Structure:**

```tsx
<form
  role="search"
  action="/buscar"          // progressive enhancement: works without JS (GET)
  method="get"
  className="flex gap-2"
>
  <div className="relative flex-1">
    <Search aria-hidden className="absolute left-3 top-1/2 -translate-y-1/2 h-4 w-4 text-muted-foreground" />
    <Input
      type="search"
      name="q"
      defaultValue={initialQuery}
      placeholder="Busca por idea o número de boletín…"
      aria-label="Buscar proyectos de ley"
      className="h-12 pl-9 text-base"
      autoFocus={autoFocus}
    />
  </div>
  <Button type="submit" className="h-12">Buscar</Button>
</form>
```

**Progressive enhancement (required):** Use a real `<form method="get" action="/buscar">` so the search works with JavaScript disabled (the WAF/SSR-first principle). The `"use client"` handler may `router.push` for a snappier transition, but the bare form must submit on its own. `name="q"` is the canonical query param.

**Empty submit guard:** If the input is empty/whitespace, do not navigate (disable submit or no-op). Never route to `/buscar?q=`.

**Accessibility:** `role="search"` on the form, `aria-label` on the input, lucide `Search` icon is `aria-hidden`. Submit button has a visible text label "Buscar" (not icon-only).

**Touch target:** Input and Button both `h-12` (48px) on the landing; on `/buscar` the persistent box may use the default `h-10` (40px) — still ≥ the inherited 44px guidance via padding.

## 4. Results Page — `/buscar`

**Route:** `app/app/buscar/page.tsx` (Server Component). Reads `searchParams.q` (untrusted input — trim, length-cap, treat as plain text; never interpolate into raw SQL — the kNN RPC parameterizes).

**URL shape:** `/buscar?q=protección+de+datos` and optional `?page=2`.

**Server flow (data layer, not UI):** trim query → boletín-pattern shortcut (`/^\d{3,6}(-\d{1,2})?$/`, same regex as `/proyecto/[boletin]`) → if match, redirect to `/proyecto/{boletin}`; else embed query with Gemini `RETRIEVAL_QUERY` (768, L2) → kNN RPC (cosine, top-K=20, distance threshold) → render result cards. All server-only; Gemini key never reaches the client.

**Layout:**

```
<main className="max-w-3xl mx-auto px-4 md:px-8 py-8 md:py-16">
  <SearchBox initialQuery={q} />                ← persistent, mt-0
  <p className="text-sm text-muted-foreground mt-4">  ← result-count line
  <section className="mt-6 space-y-4">          ← <Suspense> → result cards
    [ SearchResultCard × N ]
  </section>
  [ pagination — simple prev/next, only if > 20 results ]
</main>
```

**Result-count line copy:**
- With results: `{N} resultados para "{q}"` (N is the real count of returned rows).
- One result: `1 resultado para "{q}"`.

**Pagination:** Simple prev/next using the `Button` ghost pattern from `WeekNav` (links, not JS state). Only render when there is more than one page. No infinite scroll, no numbered pages in this phase.

**Loading:** One `<Suspense>` boundary wrapping the results list; skeleton = 4 `SearchResultCard` skeletons (see §8.2).

## 5. SearchResultCard

**Purpose:** Summarized ficha tile. Reuses `EtapaBadge`, `CamaraChip`, `ProvenanceBadge` directly. Mirrors `CitacionCard` structure.

**Anatomy:**

```tsx
<Card>
  <CardHeader className="pb-2">
    <div className="flex flex-wrap items-center gap-2">
      <EtapaBadge estado={estado} />
      <CamaraChip camara={camaraOrigen} />
      <span className="font-mono text-sm text-muted-foreground">Boletín N°{boletin}</span>
    </div>
    <h3 className="text-base font-semibold mt-1">
      <Link href={`/proyecto/${boletin}`} className="hover:text-primary hover:underline underline-offset-2">
        {titulo}
      </Link>
    </h3>
  </CardHeader>
  <CardContent className="space-y-2">
    {materia && <p className="text-sm text-muted-foreground line-clamp-2">{materia}</p>}
    <div className="pt-1">
      <ProvenanceBadge {...provenance} />
    </div>
  </CardContent>
</Card>
```

**Title is the link** to `/proyecto/{boletin}` (internal — no `target="_blank"`). The whole card need not be a link; the title link plus an optional "ver ficha →" footer link is sufficient and keeps the `ProvenanceBadge` link independently clickable.

**Materia:** `line-clamp-2`, shown as-is. Never summarized, never editorialized.

**No relevance score shown.** Do NOT display cosine distance, "% match", or rank numbers to the user — they imply a precision the system does not claim. Ordering communicates relevance implicitly.

**ProvenanceBadge:** one per card, bottom of `CardContent`. `sourceName` from `sourceLabel(origen)`; `sourceUrl` = the proyecto's own `enlace`.

**Props:**

```ts
interface SearchResultCardProps {
  boletin: string;
  titulo: string;
  materia: string | null;
  estado: string | null;        // → EtapaBadge
  camaraOrigen: "camara" | "senado";
  provenance: ProvenanceBadgeProps;   // reuse Phase 5 interface
}
```

## 6. Structured Ficha Additions — `/proyecto/[boletin]`

Add two new sections to the **existing** ficha page (`app/app/proyecto/[boletin]/page.tsx`), each in its own `<Suspense>`, placed after the existing Votaciones section. Do not disturb the existing header/timeline/votaciones blocks.

### 6.1 Idea matriz + Cuerpos legales

Inserted as a `<section id="contenido">` after the ficha header (or after votaciones — executor's call, but it must appear above "Proyectos similares").

```
<section id="idea-matriz" className="mt-12">
  <h2 className="text-xl font-semibold mb-4">Idea matriz</h2>
  <IdeaMatrizBlock ideaMatriz={...} provenance={...} />
</section>

<section id="cuerpos-legales" className="mt-12">
  <h2 className="text-xl font-semibold mb-4">Cuerpos legales afectados</h2>
  <CuerposLegalesList cuerpos={...} />
</section>
```

**`IdeaMatrizBlock` — available state.** The idea matriz is a **literal quote** extracted from the source text (CONTEXT.md: extracción literal, never abstractive). Render as a blockquote to signal it is verbatim, not editorial:

```tsx
<blockquote className="border-l-2 border-border pl-4 text-base leading-relaxed text-foreground">
  {ideaMatriz}
</blockquote>
<div className="mt-3">
  <ProvenanceBadge capturedAt={...} sourceName="BCN" sourceUrl={textoFuenteUrl} />
</div>
```

**`IdeaMatrizBlock` — "no disponible" state (honest degradation).** When the source text could not be fetched (R2 401 / no link / fetch failed), the idea matriz is absent. This is a first-class state — render it gracefully, never fabricate:

```tsx
<div className="rounded-lg border border-border bg-muted/40 px-6 py-6 text-sm text-muted-foreground leading-relaxed">
  <p className="font-semibold text-foreground">Idea matriz no disponible</p>
  <p className="mt-1">
    El texto íntegro de este proyecto no pudo obtenerse de la fuente oficial,
    por lo que no se ha extraído su idea matriz. Puedes consultar el proyecto
    completo en la fuente original.
  </p>
</div>
```

- Heading: `Idea matriz no disponible`
- Never write "próximamente" (implies a commitment). Never write an LLM-generated summary as a fallback. Never imply the proyecto has no idea matriz — only that it was not extracted.
- Use `border-border bg-muted/40` (the degraded-state styling from Phase 6 §6.2), NOT `border-destructive` — absence of extracted text is not an error.

**`CuerposLegalesList`.** Normalized list (CONTEXT.md: número de ley/código + artículos citados). Render as a simple list, each item a `Badge variant="secondary"` or a `<li>`:

```tsx
<ul className="space-y-1 text-sm">
  {cuerpos.map((c) => (
    <li key={c.id} className="leading-relaxed">
      <span className="font-medium text-foreground">{c.norma}</span>
      {c.articulos?.length ? (
        <span className="text-muted-foreground"> — {c.articulos.join(", ")}</span>
      ) : null}
    </li>
  ))}
</ul>
```

If the list is empty (none extracted or text unavailable):
```
<p className="text-sm text-muted-foreground">
  No se identificaron cuerpos legales afectados a partir del texto disponible.
</p>
```

### 6.2 Proyectos similares (kNN)

`<section id="similares" className="mt-12">`, last on the page.

```tsx
<section id="similares" className="mt-12">
  <h2 className="text-xl font-semibold mb-4">Proyectos similares</h2>
  <Suspense fallback={<SimilaresSkeleton />}>
    <ProyectosSimilares boletin={boletin} />
  </Suspense>
</section>
```

**Behavior:** kNN over the current proyecto's embedding, **excluding self** (filter out the same boletín), top ~5. Renders a `space-y-4` stack of `SearchResultCard` (reused).

**Empty / unavailable state.** If the current proyecto has no embedding (idea matriz "no disponible" → embedded on título + materia, so it usually still has a vector) or kNN returns nothing above threshold:

```
<p className="text-sm text-muted-foreground">
  Aún no hay proyectos similares para mostrar. La similitud se calcula sobre
  los proyectos ya indexados; puede ampliarse a medida que se incorporan más.
</p>
```

- Never say "no existen proyectos similares" (an over-claim about the whole corpus). Say "aún no hay … para mostrar" + the indexing caveat. Factual, no causal/affinity language.
- "Similar" here means **semantic neighborhood**, never "related agenda", "allied", or any inferred intent (guardarraíl riesgo existencial #2). The heading "Proyectos similares" plus the neutral empty copy keep this neutral.

## 8. States

### 8.1 Empty States

| Context | Trigger | Copy |
|---------|---------|------|
| Results — no matches | kNN returns 0 rows above threshold | `Sin resultados` + `No se encontraron proyectos para "{q}". Prueba con otras palabras, o ingresa un número de boletín.` |
| Results — empty query reached page | `q` missing/blank on `/buscar` | Render landing-style prompt: show `SearchBox` + hint, no result list, no error. |
| Idea matriz unavailable | text not fetched/extracted | §6.1 "no disponible" block (border-border bg-muted/40) |
| Cuerpos legales empty | none extracted | §6.1 empty paragraph |
| Proyectos similares empty | no neighbors above threshold | §6.2 empty paragraph |

Empty-results container styling: `rounded-lg border border-border bg-muted/40 px-6 py-8 text-center` (matches Phase 6 degraded container). Empty is NOT an error — never `border-destructive`, never "Oops", never "error".

### 8.2 Loading States

`<Suspense>` per async section. Shape-matched skeletons, `aria-hidden="true"`, no focusable children.

**SearchResultCard skeleton** (repeat 4× on `/buscar`):
```tsx
<Card>
  <CardHeader className="pb-2">
    <div className="flex gap-2">
      <Skeleton className="h-5 w-20 rounded-full" />
      <Skeleton className="h-5 w-16 rounded-full" />
      <Skeleton className="h-5 w-28" />
    </div>
    <Skeleton className="h-5 w-3/4 mt-2" />
  </CardHeader>
  <CardContent className="space-y-2">
    <Skeleton className="h-4 w-full" />
    <Skeleton className="h-4 w-5/6" />
    <Skeleton className="h-5 w-48 mt-2" />
  </CardContent>
</Card>
```

**IdeaMatriz skeleton:**
```tsx
<div className="space-y-2" aria-hidden="true">
  <Skeleton className="h-4 w-full" />
  <Skeleton className="h-4 w-11/12" />
  <Skeleton className="h-4 w-2/3" />
</div>
```

**ProyectosSimilares skeleton:** 3 × `SearchResultCard` skeleton (above), `space-y-4`.

### 8.3 Error States

Distinct from empty/degraded. An error is a failed embed call, kNN RPC failure, or DB error.

```
<div className="border border-destructive/20 bg-destructive/5 rounded-lg p-4 text-sm">
  Ocurrió un error al realizar la búsqueda. Vuelve a intentarlo en unos momentos.
</div>
```

**Distinction — error vs. degradation vs. empty:**
- **Error** = a call failed → `border-destructive/20 bg-destructive/5`.
- **Degraded** = idea matriz "no disponible" (source text absent) → `border-border bg-muted/40`. NOT an error.
- **Empty** = valid query, zero results → `border-border bg-muted/40`, centered. NOT an error.

Never style degradation or emptiness with destructive color.

## 10. Accessibility

All Phase 5 accessibility rules apply. Phase 7 additions:

- Landing and `/buscar`: exactly one `<h1>` per page. On `/buscar` the H1 may be visually compact (e.g. `text-xl`) or `sr-only` if the persistent search box is the visual hero — but it must exist for screen readers (`<h1 className="sr-only">Buscar proyectos de ley</h1>` is acceptable).
- `SearchBox`: `role="search"` on the form, `aria-label` on the `Input`, decorative search icon `aria-hidden`, submit button has visible text "Buscar".
- New ficha sections (`Idea matriz`, `Cuerpos legales afectados`, `Proyectos similares`) are `<h2>` under the page `<h1>` — no skipped heading levels; they sit alongside the existing `<h2>` "Tramitación" / "Votaciones".
- Idea matriz verbatim quote uses a semantic `<blockquote>`.
- Result-card title link is a real `<Link>`; the `ProvenanceBadge` "fuente oficial ↗" link remains independently focusable with its existing `aria-label` and "(abre en nueva pestaña)" affordance.
- Skeleton wrappers `aria-hidden="true"`.
- Keyboard: form submits on Enter; no keyboard trap; focus ring uses `--ring` (inherited).

Same `ProvenanceBadgeProps` interface from Phase 5 (`{ capturedAt, sourceName, sourceUrl }`). New source contexts:

| Data | `sourceName` | `sourceUrl` |
|------|--------------|-------------|
| Search result card | `sourceLabel(proyecto.origen)` → "Cámara" / "Senado" | proyecto's own `enlace` |
| Idea matriz (extracted from texto íntegro) | `"BCN"` (or the texto-fuente origin) | link to the source text (BCN `obtxml` / Senado XML link) |
| Idea matriz "no disponible" | — | No `ProvenanceBadge` in the degraded block; instead the body copy points to "la fuente original". The ficha header's existing `ProvenanceBadge` still covers the proyecto record. |

- Every displayed datum keeps fuente/fecha/enlace (brand principle). The idea matriz quote carries its own provenance (the text it was extracted from), distinct from the proyecto record's provenance.
- The "no disponible" state never carries a fabricated source; it states the text was unobtainable and defers to the official source.
- Staleness threshold unchanged (amber > 48h, `esStale`).
