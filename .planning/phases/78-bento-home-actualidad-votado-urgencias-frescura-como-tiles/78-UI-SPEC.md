**Civic-token wiring caveat (load-bearing вЂ” read before writing the bar):**
`--camara` / `--senado` are defined in `app/app/styles/civic-tokens.css` as **complete `hsl(вЂ¦)` strings** (unlike `--background` etc. which are bare HSL triples). They are **NOT** registered as Tailwind color utilities (only `--color-accent-product-soft` and `--color-identity-warn-*` are, globals.css:80-88). Therefore:
- The camara bar uses **arbitrary-var syntax**: `bg-[var(--camara)]` / `bg-[var(--senado)]`. This resolves correctly *because the token is already a full `hsl()` value* (no `hsl()` wrapper needed вЂ” wrapping would be the double-`hsl()` invalid-color gotcha 54-04).
- Do **NOT** write `bg-camara` / `bg-senado` (no such util exists) and do **NOT** add `hsl(var(--camara))` (double-hsl = invalid). If the planner prefers a named util, register `--color-camara: var(--camara)` in the `@theme inline` block (mirroring `--color-accent-product-soft`, globals.css:87) вЂ” but the arbitrary-var path is the zero-wiring default and is preferred to keep the diff presentation-only.
- `--accent-product-soft` **already** generates `bg-accent-product-soft` (globals.css:87) в†’ the urgencia chip fill uses that util directly, no new wiring.
