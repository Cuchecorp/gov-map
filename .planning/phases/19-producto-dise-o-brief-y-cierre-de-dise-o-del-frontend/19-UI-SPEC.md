### 0.3 Governing principles (INVARIANTS — bind every screen)

1. **Trazabilidad a la fuente.** Every datum shown carries source · date · link via `ProvenanceBadge`. The badge is NEVER omitted — its absence would falsely imply the datum has no source (shipped contract, `provenance-badge.tsx` §6.3).
2. **Anti-insinuación (hard).** A meeting / declaration / contract / campaign contribution and a vote NEVER share an `<article>` / `<Card>` / `<li>` / `<tr>`. Carriles are sibling `<section class="mt-12">` blocks, never nested. No causal, affinity, or score language anywhere. No correlation of money↔vote. (See §10 for the full invariant set.)
3. **MONEY stays gated.** `moneyPublicEnabled(process.env)` wraps the entire `<section>` (heading included) on the ficha; page-level `notFound()` on `/contraparte/[id]`. OFF (default) = node absent from HTML, not hidden by CSS. Stays OFF until LEGAL-01 sign-off. Design must read coherently in BOTH the OFF state (sections simply absent) and the future ON state.
4. **No parlamentario photo. No partido chip.** LEGAL-03 deny-by-default; the RPC never emits partido/rut/email. Identity is text + institutional camera badge only.
5. **No invented features or data.** Design exploits only data that already exists. "Nice but no data" → deferred, marked, never designed as real.
6. **Civic colours are data, not brand.** `--camara`/`--senado` identify the institutional chamber of a datum ONLY. They are NEVER the product accent, link colour, or general UI colour (avoids political reading).

## Spacing Scale

8-point scale (all multiples of 4). Matches Tailwind defaults and the shipped layout rhythm.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `1` / `gap-1` | Icon gaps, inline badge padding |
| sm | 8px | `2` / `gap-2` | Compact element spacing, chip gaps |
| md | 16px | `4` / `gap-4`, `mb-4` | Default element spacing, section heading→body gap |
| lg | 24px | `6` | Card padding, intra-section grouping |
| xl | 32px | `8` / `md:px-8` | Desktop horizontal page padding, layout gaps |
| 2xl | 48px | `12` / **`mt-12`** | **Carril boundary** between sibling sections (anti-insinuación frontier — LOCKED) |
| 3xl | 64px | `16` / `py-16` | Page-level vertical padding |
| 4xl | 96px | `24` / `py-24` | Landing hero vertical padding (`py-16 md:py-24`) |

**Container width:** `max-w-3xl mx-auto px-4 md:px-8` is the LOCKED reading column for all single-column surfaces (landing, ficha proyecto, ficha parlamentario, agenda). The two-column result surfaces (`/buscar`, optional ficha sidebar) use `max-w-5xl` with a `lg:grid-cols-[1fr_320px]` split (main + "Mapa de fuentes" sidebar; sidebar stacks below on mobile).

Exceptions:
- Touch targets: interactive controls (search submit, nav links, pills) have a minimum hit area of **44px** height even when visually shorter (use `py-` + `min-h-11`).
- The `mt-12` carril gap is **non-negotiable and never collapsed** between data-domain sections, even if one section is empty — collapsing it would let two carriles read as one composite block (anti-insinuación).

## Typography

Geist Sans for everything except hard metadata (Geist Mono). Exactly **4 sizes** in the type ramp + 1 display, and exactly **2 weights** (Regular 400, Semibold 600). No medium/bold proliferation.

| Role | Size | Weight | Line Height | Tailwind | Usage |
|------|------|--------|-------------|----------|-------|
| Display | 36px (mobile) → 48px (md+) | 600 Semibold | 1.1 (`leading-tight`) | `text-4xl md:text-5xl font-semibold leading-tight` | Landing hero headline only |
| Heading (h1) | 30px | 600 Semibold | 1.2 (`leading-tight`) | `text-3xl font-semibold leading-tight` | Page title (ficha header, surface h1) |
| Section (h2) | 20px | 600 Semibold | 1.3 | `text-xl font-semibold` | Carril section headings (Votaciones, Reuniones de lobby…) |
| Body | 16px | 400 Regular | 1.5 (`leading-relaxed`) | `text-base leading-relaxed` | Paragraph text, descriptions, verbatim extracts |
| Label / meta | 14px | 400 Regular | 1.4 | `text-sm` | Captions, provenance, muted helper text, pills |
| Mono (metadata) | 14px | 400 Regular | 1.4 | `font-mono text-sm` | Boletín, RUT, dates (ISO/literal), IDs, amounts verbatim |

**Display accent (LOCKED brand signature):** the hero headline contains exactly ONE italic accent phrase (the "Sin alucinaciones." pattern from TributaLab), rendered in the petrol accent colour. Example for the landing: a neutral display sentence with one italic petrol clause. Never more than one italic phrase. The accent phrase carries the confidence; the rest stays sober.

**Heading hierarchy is sacred:** h1 → h2 → h3 never skips. The ficha shell keeps h1 (page) → h2 (carril) → h3 (sub-item) valid as carriles stack (per `parlamentario/[id]/page.tsx` contract).

**Mono usage rule:** any value that is an identifier, code, date, or money amount is Geist Mono. Prose is never Mono. A money amount is rendered **verbatim** in Mono (no rounding, no formatting that alters the source value).

## Color

### 4.1 60 / 30 / 10 split

| Role | Light value (HSL) | Dark value (HSL) | Usage |
|------|-------------------|------------------|-------|
| Dominant (60%) — **warm cream/paper** | `hsl(40 33% 97%)` (`--background`, NEW — replaces clinical white `0 0% 100%`) | `hsl(222 28% 7%)` (warm-neutral dark, slightly warmer than shipped `222.2 84% 4.9%`) | Page background, hero canvas |
| Secondary (30%) — card/surface | `hsl(40 30% 99%)` cards on cream; `hsl(40 20% 93%)` muted/secondary bands | `hsl(222 24% 12%)` cards; `hsl(222 20% 16%)` muted | Cards, nav band, sidebar, table headers, pills |
| Accent (10%) — **petrol/teal** | `hsl(183 38% 26%)` (`--accent-product`, NEW) | `hsl(183 34% 46%)` | See reserved-for list below |
| Destructive | `hsl(0 72% 42%)` (`--destructive`, retuned warmer for cream) | `hsl(0 62% 45%)` | Genuine destructive/irreversible actions only (see §4.3 — essentially none in current product) |

> **Cream values rationale (Claude's discretion, now LOCKED):** `40°` hue at low saturation gives the warm paper feel of TributaLab without yellowing. Foreground text stays the shipped near-black `hsl(222 47% 11%)` for AA+ contrast on cream (contrast ratio ≈ 13:1). Dark mode warms the shipped slate slightly so cream→dark feels like the same brand.
> **Petrol rationale:** `183°` desaturated teal is institutionally neutral (not red/blue → no political reading), reads as "tool chrome," and matches the TributaLab/ischilesafe teal accent. Light-mode petrol `hsl(183 38% 26%)` on cream = contrast ≈ 7:1 (AA for normal text, AAA for large/UI).

### 4.2 Accent reserved for (EXPLICIT — never "all interactive elements")

The petrol/teal accent (`--accent-product`) is used ONLY for:
1. The search box focus ring and submit button (the hero affordance).
2. Text links (the underline + hover colour) in body content.
3. The global focus ring (`--ring` retuned to petrol) for keyboard navigation.
4. The single italic display-accent phrase in the hero headline.
5. The active state of nav items and tab underlines (e.g. source-type tabs).
6. The "CENTRAL" / relevance-group divider bar (adopted from TributaLab results), as a thin accent rule — NEVER as a score or percentage.

The accent is NOT used for: section headings, body text, card borders, badges of data, civic/chamber identity, status colours, or decorative fills.

## 8. Editorial Voice Guide (Spanish, CLOSED)

**Register:** neutral, factual, sober-confident. Chilean Spanish. Addresses a general/press audience.

**Always:**
- State facts with source, date, and link. "Según {fuente}, el {fecha}…".
- Money amounts **verbatim**, in Mono, exactly as the source publishes them (no rounding, no totals that imply a verdict).
- Dates literal and prominent ("Presentada el {fecha}" in Mono).
- Use "asociado al RUT" / "registrado en {fuente}" framing for money — never possessive ("sus contratos", "su dinero").
- Label AI-generated synthesis explicitly (modelo/scope/fuentes) and keep the original source intact alongside.
- Honest degradation: "no consultado" ≠ "consultado sin resultados" ≠ "error". Never collapse them.

**Never (banned vocabulary — checker-enforceable):**
- Causal/intention language: "porque", "a cambio de", "para favorecer", "influyó", "respondió a".
- Affinity/score/ranking-as-verdict: "afinidad", "puntaje", "score", "% de coincidencia", "el peor/mejor", "ranking de los más…".
- Conflict conclusions: "conflicto de interés", "enriquecimiento", "sospechoso".
- Composing money/lobby + vote in one sentence, clause, or UI unit.
- Marketing claims or fabricated counts ("la plataforma más completa", "miles de…").

**Tone exemplars:**
- Hero: neutral display sentence + one italic petrol accent phrase. (e.g. a confident-but-sober claim with a single italic clause, mirroring "Sin alucinaciones.").
- Trust line: "Fuente, fecha y enlace en cada dato · Sin afirmar intención ni causalidad."
- Money section heading (when ON): "Contratos del Estado asociados al RUT" (exact, no possessive).
- Identity guard: name raw + "identidad no verificada".

## 10. Anti-Insinuación Invariants (HARD — DS contract, checker-enforceable)

These are the load-bearing rules. Any violation fails the design.

1. **Carril propio:** every data domain (votos, lobby, patrimonio, dinero/contratos, financiamiento) is a sibling `<section class="mt-12">` with its own `<h2>`, `Suspense`, skeleton, and honest empty. Sections are NEVER nested.
2. **Never composite:** a meeting / declaration / contract / contribution and a vote NEVER share an `<article>` / `<Card>` / `<li>` / `<tr>`. The `mt-12` gap is the carril frontier and is never collapsed.
3. **No causal/affinity/score language** anywhere (see §8 banned list).
4. **No relevance % / no score.** Search orders by implicit relevance (raw HNSW distance ASC, server-side). The "Mapa de fuentes" and source tabs show composition/counts, never a score per result. (Adapts LegalAtlas/ischilesafe relevance bars OUT.)
5. **No "ranking de los peores."** Any ordering in the parlamentarios directory or lists is by a NEUTRAL observable fact (e.g. alphabetical, by chamber, by observable count like rebeldías presented as data), always with a `MethodologyCaveat`. Never framed as a verdict table. (Adapts ischilesafe lowest/highest framing OUT.)
6. **MONEY gate:** OFF (default) → section/page node ABSENT from HTML (not CSS-hidden). Stays OFF until LEGAL-01.
7. **Identity guard:** an entity is linked to a ficha ONLY if `estado_vinculo = confirmado`. Otherwise: raw name + `IdentityMarker`, never a link. Contrapartes are always raw text + IdentityMarker (RPC never emits contraparte_id/RUT).
8. **PII never rendered:** RUT, partido, email, family data never reach the public DOM or the LLM (LEGAL-03). Money sections render the subject (proveedor/donante) as their own subject ("Aporta:" / proveedor), never the parlamentario's RUT.
9. **AI labelled, source intact:** any AI synthesis carries modelo/scope/fuentes chips and the original source is shown intact below it (adopted from LegalAtlas/TributaLab).
10. **Attribution per dataset:** CC BY 4.0 only where the source licenses it (InfoProbidad); ChileCompra "mención de la fuente", SERVEL "términos de uso por verificar". Never blanket CC BY 4.0.
