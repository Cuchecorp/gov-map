# Phase 16 — UI Design Contract

> Visual and interaction contract for `/contraparte/[id]` — the citizen-facing
> page that aggregates MONEY facts (ChileCompra contratos + SERVEL aportes) for
> ONE contraparte that is a **persona jurídica** (empresa).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This page MIRRORS the shipped MONEY system.** It does not invent a new design
> language. Every token, badge, lane rule, honest-state and attribution string is
> reused from `contratos-de-parlamentario.tsx` + `financiamiento-de-parlamentario.tsx`
> + `parlamentario/[id]/page.tsx`. Differences are scoped to: the SUBJECT is the
> empresa (not the parlamentario), the page has TWO MONEY lanes instead of being a
> section, and the gate is enforced at PAGE level (`notFound()`), not section level.

## Page Anatomy (LOCKED — the load-bearing layout)

```
/contraparte/[id]   (Server Component, app/app/contraparte/[id]/page.tsx)

main.max-w-3xl …
├── GATE (page-level, FIRST):
│     if (!moneyPublicEnabled(process.env)) notFound();      // OFF default → 404, no heading leak
│     if (!CONTRAPARTE_ID_RE.test(id))       notFound();      // bad id → 404
│
├── HeaderSection  (RPC agregado_por_contraparte; security-definer; jurídica-only)
│     ├── if error  → throw (route error boundary, honest error UI)   // #34
│     ├── if !data  → notFound()                                      // unknown / NOT jurídica → 404
│     ├── if data.tipo_persona !== 'juridica' → notFound()            // defense in depth; natural never rendered
│     ├── h1  "{nombre de la empresa}"            (text-xl semibold)
│     ├── badge "Persona jurídica"                (neutral, civic muted; states WHY it is public)
│     └── intro line (text-sm muted): scope + "hechos públicos independientes, sin juicio"
│
├── section#contratos.mt-0          ← CONTRATOS LANE
│     h2 "Contratos del Estado en que aparece esta empresa"
│     Suspense → ContrapartContratosSection
│
├── section#aportes.mt-12           ← APORTES LANE  (mt-12 = anti-insinuación carril border)
│     h2 "Aportes de campaña registrados a nombre de esta empresa"
│     Suspense → ContrapartAportesSection
│
└── section#enlaces.mt-12           ← OPTIONAL: CONFIRMED-PARLAMENTARIO FACT LANE
      (renders ONLY if the RPC returns confirmed links; own lane, traced, no vote anywhere near it)
      h2 "Vínculos confirmados con parlamentarios"
      → ConfirmedLinksSection
```

**HARD anti-insinuación rules (LOCKED — these are honesty contracts, not cosmetics):**

1. **No vote, ever, on this page.** This page renders ZERO voting data. A money
   contraparte and a vote NEVER share a page, a `<section>`, an `<article>`, a `<li>`,
   a `<tr>`. The donation→vote correlation is forbidden by composition, not just by copy.
2. **Lanes are siblings, never nested.** Contratos and aportes live in separate
   `<section>`s split by `mt-12`. They are never merged into one list or one card.
3. **No causal / affinity language anywhere.** No "influye", "favorece", "a cambio de",
   "cercano a", "vinculado a su voto". Copy describes independent public facts: counts,
   totals-as-counts, dates, sources. "X aparece como contratista N veces en el periodo Y."
4. **Confirmed link to a parlamentario is a SEPARATE, traced fact** in its own lane —
   phrased as a registry fact ("Aparece vinculada al parlamentario {nombre} por
   {RUT-exacto / vínculo confirmado del pipeline}"), NEVER insinuating that any donation
   or contract relates to any vote. If the pipeline has no confirmed link, this lane is absent.
5. **CERO cómputo.** The UI does not sum montos, rank, compute %, or color by magnitude.
   Counts are the only aggregate ("N contratos", "N aportes") — never a money total.
   Montos render verbatim or "No publicado".

## Honest States (LOCKED — per lane, textually distinct)

Each lane derives its state server-side and renders three textually-distinct states.
A zero-row aggregation NEVER reads as "limpio" / "sin contratos ✓" / positive. Absence of
query ≠ absence of facts. This is the shipped pattern (`ContratosEstado`, `FinanciamientoEstado`).

### Contratos lane

| State | When | Copy (intent) |
|-------|------|---------------|
| `no_consultado` | no aggregation marker for this contraparte | "Aún no hemos consolidado los contratos de ChileCompra para esta empresa. Esto no significa que no existan." |
| `consultado_sin_contratos` | marker present, 0 contratos at corte | "Revisamos ChileCompra para esta empresa (corte al {fecha mono}) y no se registran contratos a esa fecha." |
| `con_contratos` | ≥1 contrato | Neutral count line: "{N} contrato(s) registrado(s)." + paginated rows |

### Aportes lane

| State | When | Copy (intent) |
|-------|------|---------------|
| `no_consultado` | no aggregation marker | "Aún no hemos consolidado los aportes de SERVEL para esta empresa. Esto no significa que no existan." |
| `consultado_sin_aportes` | marker present, 0 aportes at corte | "Revisamos SERVEL para esta empresa (corte al {fecha mono}) y no se registran aportes a esa fecha." |
| `con_aportes` | ≥1 aporte | Neutral count line: "{N} aporte(s) registrado(s)." + rows grouped by elección DESC (mirror sibling) |

### Loading (Suspense fallback)

Per-lane `Skeleton` shape-matched to the row layout (intro line + attribution line + 3
row blocks), copied from `ContratosSkeleton` / `FinanciamientoSkeleton` in the ficha page.

### Error

A real RPC/DB error THROWS to the route error boundary (#34) — it NEVER degrades to
"sin contratos" / "sin aportes". Honest error UI, sober Spanish, never "limpio".

### Gate OFF (page-level — the new behavior for Phase 16)

`moneyPublicEnabled(process.env)` is `false` (default) → `notFound()` BEFORE any heading,
RPC, or Supabase read. The page returns the standard 404 surface (mirror
`parlamentario/[id]/not-found.tsx`). **No MONEY content, no h1, no lane heading leaks into
the HTML.** The gate is enforced at the top of the page component, so the entire tree is
absent — not merely an empty section. (Distinct from the ficha, where the gate wraps a
`<section>`; here the gate wraps the whole route.)

## Row Layout (mirror the siblings, subject swapped)

The SUBJECT of every row is the EMPRESA's activity — but since the page IS the empresa,
the row foregrounds the COUNTERPARTY-SIDE fact (organismo comprador for contratos;
candidato/elección for aportes), with the empresa implicit as page subject.

**Contrato row** (`<li>` with `border-t`, `py-4`):
- `<dl>` label/value pairs, verbatim: Organismo comprador · Nombre de la orden · Monto
  (→ "No publicado", never a non-monto) · Fecha de la orden (mono) · Código de la orden (mono)
- `fecha_corte` line ("Consolidado, corte al {fecha mono}.")
- `ProvenanceBadge` (sourceName `ChileCompra`, capturedAt, enlace) — MANDATORY, right-aligned.

**Aporte row** (grouped by `Elección:` header, mirror sibling):
- `Elección:` is load-bearing and present on EVERY row (defense in depth) — an aporte from
  a prior election carries its periodo and the amber "candidatura anterior" caveat is N/A
  here (the page subject is the empresa, not a candidate), but the periodo is still shown.
- `<dl>`: Elección · Candidato (the recipient, verbatim, as a SEPARATE muted fact —
  "Registrado a la campaña de {candidato}", never fused with a vote) · Fecha del aporte
  (mono) · Monto ("No publicado" if null) · Tipo de aporte
- `donante` RUT is NEVER rendered (the donante here IS the empresa = page subject; its RUT
  is not projected by the RPC — Ley 21.719 discipline carries over).
- `ProvenanceBadge` (sourceName `SERVEL`, capturedAt, enlace) — MANDATORY.

**Confirmed-parlamentario lane row** (only if present):
- "Esta empresa aparece vinculada al parlamentario {nombre} ({vínculo confirmado})."
- Its own `ProvenanceBadge`/source for the link. NO vote, NO money fact composed into this
  sentence. It states the link as an independent registry fact.

## Pagination

Server-driven, `PAGE_SIZE = 20` per lane (mirror siblings). Query params namespaced per
lane: `?contratosPage=` and `?aportesPage=` with `#contratos` / `#aportes` anchors.
`Anteriores` / `Siguientes` links, `text-primary underline`, `min-h-[44px]`. "Página N de M"
mono, muted.

## Reaching the route (listing — minimal / deferrable)

CONTEXT §Claude's Discretion leaves "cómo listar contrapartes para llegar a la ruta" open.

**Decision for MVP:** the route is **directly addressable** (`/contraparte/[id]`) and is
reached primarily as a future link from a contrato/aporte row (the proveedor/donante name
could become a link in a later phase). A standalone listing/search at
`app/app/contraparte/page.tsx` is **DEFERRED** — not required to satisfy MONEY-05. If a
minimal listing is built, it must itself be behind `moneyPublicEnabled()` (else `notFound()`)
and list ONLY `tipo_persona = 'juridica'` contrapartes (name + neutral count), no money
totals, no ranking. **Recommended: ship the `[id]` page first; defer the listing.**

`id` validation: introduce `CONTRAPARTE_ID_RE` in `@/lib/buscar` mirroring
`PARLAMENTARIO_ID_RE` (e.g. `rut_proveedor` / `donante_id` shape — exact form is RPC-key
discretion per CONTEXT). Validate BEFORE any DB read; bad id → `notFound()`.
