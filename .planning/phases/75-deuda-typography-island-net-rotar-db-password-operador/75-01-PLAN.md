<objective>
Align the `/red` relation-diagram island `.net-*` typography to the design system by swapping every raw-rem `font-size` literal in the `.net-*` block of `app/app/globals.css` to the system's canonical Tailwind type tokens (`var(--text-base)` = 16px, `var(--text-sm)` = 14px, `var(--text-xs)` = 12px), PIXEL-PRESERVING, and add a source-level guard test so the ad-hoc-size debt cannot silently return (DEBT-05).

Tailwind v4 here has NO custom `fontSize` override, so the design-system steps ARE the Tailwind defaults, exposed as `--text-*` theme vars — referencing them makes the island consume the contract with ZERO rendered-pixel change.

Output: `globals.css` `.net-*` `font-size` declarations referencing `--text-*` tokens (except the intentional `.net-chip` 11px); a guard test asserting no raw-rem `font-size` survives in `.net-*` besides `.net-chip`.
</objective>
