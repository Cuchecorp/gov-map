## Placement Decision (the load-bearing decision) — LOCKED

**The chart lives at the TOP of the collapsed Votaciones DETAIL (capa-2), NOT in capa-1.**

New sub-block rendered as the FIRST child of `VotosSection` (inside the existing
`DetalleColapsable`, above "Cómo votó"). Capa-1 (`VotosCapa1`: 5 Mono figures +
"Cómo votó" stacked bar) is unchanged byte-for-byte.

Rationale (four independent reasons, all LOCKED constraints):

1. **Cognitive role (Shneiderman overview→detail).** Capa-1 is the PREATTENTIVE
   layer — big Mono figures parsed at a glance. A time-series chart must be *read*,
   not glanced; it is a details-on-demand object by nature. It belongs to capa-2.
2. **F46 gramática is LOCKED.** 55-UI-SPEC Component Inventory: capa-1 mini-visuals =
   "CSS on existing tokens, **no chart lib**"; Recharts charts "**live in the detail**"
   (PatrimonioSection). Putting a Recharts island in capa-1 would break the
   established pattern the checker validates against.
3. **First-viewport LOCKED (F55).** A full evolution chart (~h-72) in the always-visible
   band pushes the Lobby carril far below the fold and adds a non-preattentive object
   to the scan band → the capa-1 becomes less scannable. The CONTEXT constraint "el
   chart NO puede hacer la capa-1 menos escaneable / el primer viewport de F55 no se
   degrada" forbids this. Placing it inside the collapsed detail keeps the first
   viewport byte-identical to F55.
4. **Client-island containment.** The chart is a `"use client"` Recharts island
   (`red-graph`/`patrimonio-chart` pattern). VotosSection is already the detail
   subtree; the island stays out of the always-shipped preattentive path and never
   imports the server-only Supabase client (no-leak F45).

Discarded alternative: "chart compacto en capa-1". Rejected for reasons 1–4 above.

## The Chart — Contract

### Component
- New client island `app/components/votos-chart.tsx` (`"use client"`), mirroring
  `patrimonio-chart.tsx` exactly: receives ONLY a flat serialized `VotoPeriodo[]`
  (strings + numbers) computed server-side; imports Recharts + a `type` only; never
  imports the server-only Supabase client or the Server Component runtime.
- New pure server-side aggregator (testable, exported) that buckets the
  ALREADY-FETCHED `votos` rows (each `VotoFichaConMateria` carries `fecha` +
  `seleccion`) into time periods with neutral per-sense counts. NO new fetch, NO new
  RPC — the temporal dimension is already in `votos_de_parlamentario` (proven by
  `resumenDeArco`/`mesAnio` already parsing `new Date(fecha)`).

### Chart type — LOCKED (F46 honesty rule carried forward)
- **Stacked `BarChart`, one DISCRETE bar per time bucket.** NEVER a line or area
  chart: a connected series over time would insinuate a "trend"/"trajectory" of
  behaviour — anti-insinuación HARD, identical to the F46 "stacked-NO-line" rule.
- Each bar = one time bucket; segments stacked in the LOCKED vote order
  (si → no → abstención → pareo → ausente), `stackId="votos"`.
- `YAxis allowDecimals={false}` (counts only); `XAxis` = period label (Mono ticks).

### Granularity — trimestre (default; Claude's discretion window)
- **Bucket by calendar quarter (`AAAA-Tn`, e.g. "2024 · T2").** Rationale: the data is
  sparse per parlamentario (dataset = 133 votaciones / 186 parlamentarios over the
  legislatura); monthly buckets would fabricate many near-empty columns and imply a
  denser cadence than exists. Quarterly is the honest granularity: enough buckets to
  show distribution across the period without inventing a smooth curve.
- Buckets are derived ONLY from periods that actually contain ≥1 vote — NO empty
  buckets fabricated between the first and last quarter (a gap is a fact, not a zero).
- If only ONE bucket has data → render the single bar honestly (one quarter is a fact,
  not a trend). Never pad to fake a series.

### Colors — data-encoding, NEVER the 10% petróleo (LOCKED)
Reuse the EXISTING vote semantic colors (single source with `VotosCapa1` SEGMENTO so
the chart segments never desync from the capa-1 stacked bar). Recharts fills need
explicit values (like `patrimonio-chart.tsx`); map 1:1 to the capa-1 `bg-*` classes:

| Sense | capa-1 class | Recharts fill |
|-------|--------------|---------------|
| si (a favor) | `bg-green-500` | `hsl(142 71% 45%)` |
| no (en contra) | `bg-red-500` | `hsl(0 84% 60%)` |
| abstención | `bg-amber-400` | `hsl(43 96% 56%)` |
| pareo | `bg-slate-400` | `hsl(215 20% 65%)` |
| ausente | `bg-slate-300` | `hsl(213 27% 84%)` |

Petróleo (`--accent-product`) is **PROHIBITED** in this chart (reserved for cruces/
drill-down, F55 §Color). These colors ENCODE data; they are not brand accent.

### Dimensions / layout
- Container `h-72 w-full` (parity with `patrimonio-chart`; inside the detail there is
  room). `ResponsiveContainer width="100%" height="100%"`.
- `Legend` with NOUN labels (mirror OPCION_LABEL: "A favor / En contra / Abstención /
  Pareo / Ausente"). `Tooltip` shows the period label + per-sense counts (Mono).
- `role="img"` + `aria-label` = "Número de votos por trimestre y sentido del voto"
  (a11y: counts never conveyed by color alone — Legend + Tooltip carry text).

## Copywriting Contract (es-CL, factual, anti-insinuación LOCKED)

| Element | Copy |
|---------|------|
| Chart sub-block heading (`<h3>`) | **"Cuándo votó"** |
| Caption / caveat under chart | **"Cada barra agrupa las votaciones de un trimestre por sentido del voto. No representa una tendencia."** |
| Provenance caption (Mono for range) | **"Votaciones confirmadas · {rango de trimestres}"** (source/date already on each detail row via ProvenanceBadge; the chart adds the aggregate range) |
| Empty — votes exist but NO parseable dates | **"Las fechas de estas votaciones aún no permiten agruparlas por período."** (honest text, NEVER a fabricated zero bar) |
| Empty — parlamentario sin votos | Handled upstream: the whole detail does not render (capa-1 shows honest state). The chart is NEVER rendered with an empty/zero series. |
| Single-bucket case | Render the one bar; no special copy, no "insufficient data" scolding — one quarter is a fact. |
| Primary CTA (this phase) | **None new.** The chart lives inside the existing disclosure; its trigger is the existing **"Ver detalle ({N})"** on the Votaciones carril. |
| Destructive confirmation | **None** — no destructive actions. |

Anti-insinuación invariants (LOCKED, negative-match enforced — banned-vocab test must stay green):
- ZERO ranking/superlative language. **"el más ausente" / "el que más falta" and any
  superlative or comparative-across-parlamentarios framing is PROHIBITED** (negative-match).
- ZERO causal vocabulary (influencia, conexión, presión, a cambio de, gracias a, score,
  ranking, tendencia-afirmada, etc.). The word "tendencia" appears ONLY in the caveat
  that DENIES it ("No representa una tendencia").
- Counts are neutral facts side by side; the chart never composes a vote with a meeting
  or any other domain.
- Pareo, if present, is shown as a factual segment with its plain label ("Pareo"),
  never explained as strategy or intent.
- Empty/gap = honest fact; NEVER a fabricated bar at zero (LOCKED).
