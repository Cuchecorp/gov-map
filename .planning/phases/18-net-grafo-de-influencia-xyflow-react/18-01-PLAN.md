## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon → Postgres (RLS) | anon no debe leer entidad/arista directo; solo el RPC security definer proyecta datos PII-safe |
| RPC params → SQL | p_id/p_depth/p_tipos/fechas son input no confiable; clamp y filtros en-SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Information Disclosure | subgrafo_red projection | mitigate | proyectar solo id/nombre/camara; pgTAP introspecciona cuerpo y salida (sin partido/rut) |
| T-18-02 | Elevation of Privilege | anon read de entidad/arista | mitigate | RLS deny-by-default (enable RLS + 0 policies + revoke all); pgTAP afirma 0 filas a anon |
| T-18-03 | Denial of Service | recursive CTE walk | mitigate | clamp p_depth 1..2 in-SQL; índices en extremo_a/b/tipo; semilla obligatoria (no seedless) |
| T-18-04 | Information Disclosure | enumeración seedless | mitigate | RPC exige p_id; no se expone variante whole-graph (espejo WR-03/0025) |
</threat_model>
