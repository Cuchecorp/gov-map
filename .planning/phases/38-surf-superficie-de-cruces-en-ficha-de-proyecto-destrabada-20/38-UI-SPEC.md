## Copywriting Contract

Anti-insinuación conditions of the signed sign-off (`SIGNOFF-senales-voto.md`) are HARD
contract items — every copy string below is written to satisfy them.

| Element | Copy |
|---------|------|
| Section heading `<h2>` (petróleo) | **"Cruces con el sector del proyecto"** |
| Capa-1 count suffix (3-state honest, Mono) | **"{N} parlamentarios"** when N>0; **"sin registros"** when 0; **"—"** when no ingesta. Never a fabricated digit. |
| Capa-1 frame / intro (factual, non-causal) | **"Parlamentarios que registran un voto a favor de este proyecto y, por separado, audiencias de lobby registradas en el sector del proyecto. Cada hecho se muestra tal como lo publica la fuente oficial; el enlace de cada evidencia apunta al registro original. Un cruce solo yuxtapone hechos fechados; no afirma intención ni causa."** |
| Primary CTA (the único énfasis) | **"Explorar los {N} cruces"** — petróleo primary trigger (`DetalleColapsable triggerVariant="primary" triggerLabel`), toggles to **"Ocultar detalle"** when open |
| Per-parlamentario evidence heading (detail, neutral count) | **"{N} reuniones con gestores del sector {sector}"** (singular: "1 reunión"); degrade for unknown señal type → **"{N} registros en el sector {sector}"** — NEVER fabricate a meeting verb |
| Vote-side factual line (neutral, no causal composition) | **"Votó a favor de este proyecto"** — rendered as a SEPARATE factual line, never composed with the meeting into one causal sentence |
| Evidence meta line (per lobby item) | **"Reunión registrada el {fecha}"** (Mono date; omitted if source lacks the date — honest-state) |
| Caveat anti-causal (LOCKED, exactly 1× in this section) | **"La coincidencia temporal no implica relación entre la reunión y el voto."** (reused verbatim from `CrucesCapa1`) |
| Empty state (RPC present, 0 rows) | **"Aún no se registran parlamentarios con cruces en el sector de este proyecto en las fuentes consultadas."** (a void is a FACT, never "limpio"/"transparente") |
| Degrade pre-apply (RPC absent → PGRST202) | **No copy — the section renders `null`.** The `mt-12` wrapper persists (frontier), the `<h2>` and caveat live INSIDE the component so nothing orphans. |
| Error state (real DB/network error) | Existing `error.tsx` boundary (#34 throw). No new copy. A real error is NEVER degraded to the empty state (that would fabricate a false exoneration). |
| Destructive confirmation | **None** — no destructive actions in this phase |

Anti-insinuación (LOCKED, negative-match enforced by the banned-vocab test):
- ZERO causal / affinity vocabulary in any new string. Banned lexicon includes **"a cambio
  de", "influyó", "gestionó", "presionó", "afinidad", "porque", "antes de votar", "a
  favor de la reunión", "cercano a", "vinculado a", "score", "ranking", "conflicto de
  interés"**. The banned-vocab negative-match test must stay green.
- Counts are NEUTRAL ("N reuniones · M votos"), never framed as a relationship, ranking,
  or verdict.
- The vote fact and the meeting fact are presented **side by side as dated coincidences**,
  never composed into one sentence of causation (dossier 17 §2 condition).
- Caveat appears **exactly once** in the section (sign-off condition 1: 1×/sección).

## Layout & Interaction Contract (NEW section wiring)

Inherits the F55 rail + capa-1 + progressive-disclosure pattern verbatim. This section
declares only the wiring of the new carril into `app/app/proyecto/[boletin]/page.tsx`.

**Section mount (server):**
```
<section id="cruces" className="mt-12 scroll-mt-6">
  <Suspense fallback={<CrucesSkeleton />}>
    <CrucesSection boletin={boletin} />   {/* h2 + caveat live INSIDE the component */}
  </Suspense>
</section>
```
- Placement in DOM order: **after `#lobby-tramitacion`, before `#idea-matriz`** — it
  synthesizes the vote + sector-lobby facts, so it follows both. (Discretion, documented.)
- The `<h2>` and the caveat live INSIDE `CrucesSection` (not in the page) so the degrade
  path returns `null` with no orphan heading — same idiom as `LobbyEnTramitacionSection`.

**Rail entry (`ProyectoRail.navEntries`, server):**
- Add `{ id: "cruces", label: "Cruces", marker: "◆", accent: true }` (petróleo diamond +
  label, mirroring F55 parlamentario "Cruces ◆ {N}"). Include the honest count when the
  server can cheaply derive it (>0); omit the count otherwise. If the rail primitive has
  no marker/accent prop yet, the planner adds a minimal one — it must NOT leak petróleo to
  the other rail entries.
- Rail entry present follows the same pattern as `lobby-tramitacion`: the entry exists and
  the `<section>` wrapper always renders; the scrollspy target `#cruces` is always in the
  DOM even when the content degrades to `null`.

**Progressive disclosure (inherited, no new mechanism):**
| Layer | What | Mechanism |
|-------|------|-----------|
| Capa 1 (overview) | Petróleo frame + `<h2>` + neutral count/chips + caveat + primary trigger. Always visible. | Server-rendered (`CrucesCapa1`-style), always in DOM |
| Capa 2 (detail) | Per-parlamentario evidence groups (name link + "votó a favor" line + neutral meeting count + evidence list). | Existing `DetalleColapsable` (`triggerVariant="primary"`, `forceMount`), client toggle over ALREADY-FETCHED server data — NO lazy-fetch, NO extra RPC |
| Capa 3 (traceability) | `ProvenanceBadge` (fuente · fecha · enlace) on every lobby evidence row. | Always present, Mono/muted/`text-xs`, never competes for attention |

- Server Components by default; the disclosure island receives the section as `children`
  and NEVER imports the Supabase server client (Camino A `service_role` never reaches the
  browser — LOCKED F45 no-leak contract).

## Departure from the `/parlamentario` cruces analog (LOCKED — the ONE difference)

The `/parlamentario` cruces surface renders the **contraparte** (a private third party) as
**plain, non-linked crude text + `IdentityMarker`** (LOCKED 52-03 / dossier 41 §7). That
rule protects a third party and does NOT apply here.

In THIS section the subject IS the **parlamentario público**:
- Render the parlamentario name via **`formatNombre` (F54)** as a **LINK to
  `/parlamentario/[id]`** (petróleo primary text link, `underline underline-offset-4`).
- Projection is **PII-safe via `parlamentario_publico`** — the RPC emits
  `nombre_normalizado` / `id` and NEVER `rut` / `partido` / `email` (sign-off condition 4;
  lockdown-guard allowlist enforced).
- If the RPC ever surfaced a lobby **contraparte** name inside the evidence (a private
  third party), that name stays plain, non-linked, crude text + `IdentityMarker` — the
  52-03 rule still governs contrapartes, only the parlamentario subject is linked.
