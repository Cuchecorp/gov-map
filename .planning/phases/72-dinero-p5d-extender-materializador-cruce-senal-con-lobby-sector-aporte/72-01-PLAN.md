- Fuente de SECTOR = la clasificación de lobby: `lobby_contraparte.sector_id` (0038). Un parlamentario cuenta en la señal SOLO si tiene AMBOS: (a) contratos ChileCompra por RUT-exacto (parlamentario_id confirmado) Y (b) contrapartes de lobby con sector conocido (doblemente fail-closed).

- El token NO está reservado en el CHECK real (solo en un comentario). El CHECK real es `check (tipo_senal in ('lobby_sector'))` (0039:50). La migración DEBE ampliarlo a `('lobby_sector','lobby_sector_aporte')`.
- El "vía RUT" se satisface SIN mencionar `rut` en el cuerpo: se une por `contrato.parlamentario_id` CONFIRMADO (ese FK nació SOLO por RUT-exacto aguas arriba, Phase 70/reconciliar-contrato). El sector viene de las contrapartes de lobby CONFIRMADAS del MISMO parlamentario. Camino (a) de Pitfall 3 en RESEARCH.
- Topología del JOIN (verificado leyendo los schemas): `lobby_contraparte` NO tiene columna RUT (0021:83-85, minimización). Por tanto la arista sector↔dinero NO es por RUT crudo entre tablas: es por `parlamentario_id` común — contratos del parlamentario × sectores de sus lobby-contrapartes clasificadas. Nunca name-match de empresa.
- La señal rinde 0 filas HOY (RUT-01 0% + backfill ChileCompra pendiente). Eso es CORRECTO por requisito (vacío honesto por construcción), no un bug.

Función a re-emitir (0039:81-122) — cuerpo actual, la rama lobby (líneas insert 91-120) se conserva VERBATIM:
```
create or replace function cruces.materializar_cruces()
returns void language plpgsql security definer set search_path = '' as $$
begin
  delete from public.cruce_senal;               -- FULL REBUILD, ÚNICO delete
  insert into public.cruce_senal
    (parlamentario_id, sector_id, tipo_senal, conteo, evidencia, dataset, origen, fecha_captura, enlace)
  select a.parlamentario_id, c.sector_id, 'lobby_sector', count(*), jsonb_build_object(...), 'lobby', min(a.origen), now(), min(a.enlace)
  from public.lobby_audiencia a
  join public.lobby_contraparte c on c.identificador = a.identificador
  where a.estado_vinculo = 'confirmado' and a.parlamentario_id is not null and c.sector_id is not null
  group by a.parlamentario_id, c.sector_id;
  -- AQUÍ va la NUEVA rama lobby_sector_aporte (segundo insert...select).
end;
$$;
```
