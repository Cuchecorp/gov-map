## MEDIO

### #1 — RobotsGuard fetcha robots.txt sin pasar por `assertAllowedUrl` (SSRF ciego)
**Ubicación:** `packages/ingest/src/robots.ts:80-101` · **categoría:** security · **contested** (1 confirma / 1 refuta)

**Descripción.** `loadRobots` construye `robotsUrl = ${origin}/robots.txt` y lo fetcha directo. El allowlist se valida recién en el paso 4 (`fetcher.ts:97 assertAllowedUrl`), pero `robots.isAllowed(spec.url)` corre en el paso 2 (`base-connector.ts:127`) con la URL cruda. Un conector que emita un `spec.url` con host interno (loopback, `169.254.169.254`, `kong:8000`, RFC1918) dispara un GET a ese host **antes** de que el allowlist lo rechace — agujero en la contención SSRF que `allowlist.ts:13-15` promete.

**Evidencia.**
```
// base-connector.ts:127 — robots.isAllowed(spec.url) ANTES de assertAllowedUrl
// robots.ts:81
const robotsUrl = `${origin}/robots.txt`;  // fetch sin assertAllowedUrl
```

**Fix.** Inyectar `AllowlistOptions` en `RobotsGuard` y llamar `assertAllowedUrl(robotsUrl, allowlist)` antes del fetch (o gatear en `isAllowed`). Acotado: solo GET, path fijo, UA identificatorio — pero rompe el invariante deny-by-default. Severidad medium justa.

### #2 — upsert por `id` puede sobreescribir `estado='confirmado'` previo
**Ubicación:** `packages/identity/src/writer-supabase.ts:65-75` · **categoría:** idempotency · alegada high → **medium**

**Descripción.** `upsert(rows, { onConflict:'id', ignoreDuplicates:false })` actualiza TODAS las columnas, incluida `estado`. Las filas llegan con `estado='no_confirmado'` (`seeder.ts:156`). Si la DB ya tenía `confirmado` de un `--promote` previo, el upsert lo pisa. `preserveEstado` (`seed-cli.ts:287-307`) repara el snapshot en memoria pero **no re-escribe la DB**. No hay trigger protector sobre `parlamentario.estado` (sí lo hay sobre `vinculo_identidad` en 0007).

**Evidencia.**
```ts
await this.client.from(this.table)
  .upsert(rows, { onConflict: "id", ignoreDuplicates: false });
```

**Fix.** Excluir `estado` del update-on-conflict, o añadir trigger anti-regresión análogo al de `vinculo_identidad`. **No es high** porque el snapshot git es la fuente autoritativa (ID-09) y se auto-cura en el próximo `--promote`; la divergencia es transitoria en un store derivado.

### #3 — `resolverRevision` + `upsertVinculo` + `appendAudit` no son atómicas
**Ubicación:** `packages/adjudication/src/revisor-cli.ts:201-251` · **categoría:** error-handling

**Descripción.** Tres llamadas PostgREST independientes sin transacción. Si `resolverRevision` commitea (`estado→'confirmado'`) y luego `upsertVinculo` o `appendAudit` fallan por red/Supabase, el caso queda resuelto sin vínculo ni audit — rompe la invariante "audit en cada decisión" (T-04-11) en la compuerta humana. El reintento aborta antes (guard `caso.estado !== 'pendiente'`), dejando el huérfano sin reparación por el camino normal.

**Evidencia.**
```ts
const { afectadas } = await w.resolverRevision(...);
if (afectadas === 0) throw ...;
// upsertVinculo / appendAudit pueden lanzar a continuación sin rollback
```

**Fix.** RPC Postgres transaccional (`rpc('resolver_identidad', {...})`) con UPDATE+UPSERT+INSERT en una transacción + job de reconciliación que detecte casos resueltos sin audit. Fail-safe asimétrico (deja caso huérfano, no vínculo corrupto), pero estado durable inconsistente en subsistema crítico.

### #4 — `toIso()` desplaza el día al serializar (medianoche local → UTC)
**Ubicación:** `packages/tramitacion/src/fecha.ts:45-47` · **categoría:** correctness · alegada high → **medium**

**Descripción.** `parseFechaCL` construye `new Date(yyyy, mm-1, dd)` (medianoche **local**) y `toIso()` llama `toISOString()` (UTC). El ISO almacena una hora fantasma (`T04:00Z` en Chile) y, en zonas UTC-positivas, corre la fecha de calendario al día anterior. El propio test `fecha.test.ts:40` (`/^2026-06-0[23]T/`) tolera el shift. Impacta los upserts y el discriminador de votación por fecha; el ordenamiento del timeline NO (usa `getTime()` del Date local).

**Evidencia.**
```ts
const d = new Date(yyyy, mm - 1, dd);
...
return d.toISOString();
```

**Fix.** `new Date(Date.UTC(yyyy, mm-1, dd))` → `T00:00:00Z` limpio e independiente del TZ del proceso. **No es high** porque producción corre en UTC (Edge Functions / GitHub Actions); el síntoma se materializa solo en TZ negativa (máquina del dev) y el ejemplo del finder estaba invertido.

### #5 — `BOLETIN_RE` demasiado permisiva captura números no-boletín
**Ubicación:** `packages/agenda/src/parse-camara-citaciones.ts:26` · **categoría:** correctness

**Descripción.** `/\bN?[°ºo]?\s?(\d{3,5})[.\s]?-?(\d{1,2})\b/gi` tiene prefijo y separador opcionales, así que cualquier número de 4-5 dígitos se parte en falso boletín: `"2026"→"202-6"`, `"ley 21000"→"2100-0"`, `"12345"→"1234-5"`. Contamina `citacion_punto.boletin` — *la llave de cruce* — y produce vínculos falsos con la ficha. El schema (`model.ts:58 z.string().nullable()`) y el test (`/^\d{3,5}-\d{1,2}$/`) no filtran la basura. Además mis-parsea boletines legítimos con punto de miles (`"N° 16.234-07"→"234-07"`).

> Nota: los ejemplos concretos del finder (`"el -2"`, `"123"`) NO disparan el bug; el mecanismo real son números pelados de 4-5 dígitos.

**Evidencia.**
```ts
const BOLETIN_RE = /\bN?[°ºo]?\s?(\d{3,5})[.\s]?-?(\d{1,2})\b/gi;
```

**Fix.** Exigir guión separador obligatorio, p.ej. `/\bN?[°º]?\s*(\d{4,5})-(\d{2,3})\b/gi`, contemplando el punto de miles (`16.234-07`). Validar contra golden set con años, artículos y porcentajes.

### #6 — Fecha de Cámara en español persiste a columna `timestamptz`
**Ubicación:** `packages/agenda/src/parse-camara-citaciones.ts:69` · **categoría:** correctness

**Descripción.** `parseFechaEsCl` retorna `normWs(texto)` (texto crudo en español) cuando el regex no casa. Eso llega a `citacion.fecha` (`timestamptz` en `0010_agenda.sql:23`); el schema `z.string()` no lo filtra. El upsert es un lote de hasta 500 filas: una sola fecha inválida aborta el lote y el catch por-semana (`ingest-run.ts:128-139`) **descarta la semana completa**, incluidas filas válidas. Asimétrico con el parser hermano del Senado (`parseFechaDmy` retorna `null` → descarta la fila).

**Evidencia.**
```ts
const m = t.match(/(\d{1,2})\s+DE\s+([A-ZÁÉÍÓÚÜÑ]+)\s+DE\s+(\d{4})/);
if (!m) return normWs(texto);
```

**Fix.** Retornar `null` y que el caller descarte la fila (espejando el Senado). Latente: el fixture casa, se dispara solo ante drift del HTML gubernamental (históricamente volátil).

### #7 — `upsertFicha('embebido')` antes de `upsertEmbedding` → boletín huérfano
**Ubicación:** `packages/fichas/src/pipeline.ts:164-166` · **categoría:** idempotency · alegada high → **medium**

**Descripción.** Se escribe la ficha con `estado:'embebido'` y luego el embedding, en dos upserts PostgREST a tablas distintas sin transacción. Crash/timeout entre ambos deja `proyecto_ficha.estado='embebido'` sin fila en `proyecto_embedding`. La reanudación filtra por `estado='pendiente'`, así que el boletín nunca se completa y desaparece silenciosamente de la búsqueda semántica. Solo `--reembed` lo rescata (override manual).

**Evidencia.**
```ts
await opts.writer.upsertFicha([fichaRow]);
await opts.writer.upsertEmbedding([embRow]);
embebidos += 1;
```

**Fix.** Invertir el orden (embedding primero, luego marcar `'embebido'`) — estrictamente mejor: un embedding huérfano es imposible y una ficha `'pendiente'` sin embedding se reintenta sola. O RPC transaccional. **No es high**: ventana estrecha, recuperable con `--reembed`.

### #9 — `sourceUrl` sin validación de protocolo → href `javascript:`/`data:`
**Ubicación:** `app/app/buscar/page.tsx:123-127` · **categoría:** security

## BAJO

### #10 — `fecha_captura` no valida ISO 8601 *(contested)*
`packages/core/src/parlamentario.ts:123` · correctness. `z.string().min(1).max(MAX_CAMPO)` acepta `'mañana'`. Todos los productores reales derivan de `new Date().toISOString()`, así que es defensa-en-profundidad. **Fix:** `.datetime()` o `refine(v => !isNaN(Date.parse(v)))`.

### #11 — `fold()` no normaliza apóstrofes ni guiones
`packages/core/src/nombre.ts:51-59` · correctness. Solo reemplaza `.`/`,`. `O'Higgins`/`Ohiggins`/`O-Higgins` no colapsan al mismo token de blocking. Edge ortográfico estrecho sin parlamentario afectado conocido. **Fix:** ampliar a `/[.,'\-]/g`.

### #12 — Etapa blocking-vacío escribe `metodo='llm'` (dup de #18) *(contested)*
`packages/adjudication/src/pipeline.ts:125-140` · correctness. Decisión 100% determinista etiquetada `'llm'` sin invocar al LLM. Contamina el audit trail. **Fix:** `metodo='determinista'` (ya permitido por el CHECK de 0006).

### #13 — `DriftDetector.alert` traga error de insert sin log
`packages/ingest/src/drift.ts:74-86` · error-handling. Catch vacío; si Supabase está degradado el evento de drift se pierde sin rastro (contradice FND-04 y el docstring del módulo). **Fix:** `console.warn('drift_alert insert falló:', e)` sin relanzar.

### #14 — `embed()` omite compuertas RUT/sensibilidad *(contested)*
`packages/llm/src/providers/gemini-embeddings.ts:75-104` · security. DeepSeek/MiniMax llaman `assertNoRutInLlmInput`/`assertSensitivityAllowed`; Gemini no. Decisión documentada (embedder = solo texto público) y no explotable hoy (callers reales no envían PII), pero falta defensa-en-profundidad. **Fix:** añadir las compuertas a la interfaz `EmbeddingProvider`.

### #15 — Embed sin backoff ni `RetryableError`/`FetchError` *(contested)*
`packages/llm/src/providers/gemini-embeddings.ts:106-109` · error-handling. Lanza `Error` genérico ante cualquier `!res.ok`. El backoff vive en pgmq por diseño, pero debería lanzar `RetryableError`(429/5xx) vs `FetchError`(4xx) como `fetcher.ts:41-111` para que el worker distinga. **Fix:** alinear con el patrón `RetryableError`/`FetchError`.

### #16 — `SENADO_PERIODO` hardcodeado con año
`packages/identity/src/parse-senado.ts:32` · tech-debt. `'senado-vigente-2026'` es clave de blocking, duplicada en `reconciliar-senado.ts:39` sin import compartido. **Fix:** símbolo compartido único o etiqueta sin año. Fail-closed seguro.

### #17 — `preserveEstado` hereda estado por firma colisionante *(contested)*
`packages/identity/src/seed-cli.ts:287-307` · correctness. Firma `camara|periodo|nombre_normalizado` (sin materno). Un nuevo parlamentario con mismo paterno+nombres en la misma cámara puede heredar `confirmado` del saliente. WARN solo en la dirección degradante. Contenido por el gate separado `vinculo_identidad.estado`. **Fix:** WARN + no auto-preservar `confirmado` cuando firma coincide pero `id` no.

### #18 — `metodo='llm'` en vínculo no_confirmado sin LLM
`packages/adjudication/src/pipeline.ts:124-130` · correctness. Mismo defecto que #12. **Fix:** `metodo='determinista'`.

### #19 — Duplicación ilimitada de filas `no_confirmado` *(contested)*
`packages/adjudication/src/writer-revision.ts:161-173` · idempotency. El índice único parcial de 0006 aplica solo `WHERE parlamentario_id IS NOT NULL`; cada re-proceso de una mención huérfana inserta una fila nueva → O(días) al cablear ingesta diaria. **Fix:** índice único parcial `WHERE parlamentario_id IS NULL` + `onConflict` con ignore/merge.

## Guardarraíles confirmados (intactos)

Durante la revisión, los validadores confirmaron repetidamente que los invariantes existenciales del proyecto **siguen enforced** — varios hallazgos fueron rebajados precisamente porque estos los contienen:

- **Identidad LOCKED:** `vinculo_identidad.estado='confirmado'` es el gate público de link (3 capas, triggers de 0007); separado del `parlamentario.estado` de la maestra. Un `confirmado` mal heredado (#2, #17) no auto-vincula votos/lobby.
- **Snapshot git autoritativo (ID-09):** la verdad del estado confirmado vive en git, no en la DB → contiene #2 y #17.
- **Fail-closed del matcher:** mismatch de período (#16, #22) degrada a `no_confirmado`/`sin-candidato`, **nunca** a un match equivocado. No dispara el riesgo existencial #1 (publicar match errado).
- **SSRF deny-by-default a nivel host:** `assertAllowedUrl` sigue bloqueando metadata del cloud y servicios internos en el path de fetch principal; #1 y #20/#27 son grietas acotadas en bordes (pre-fetch de robots / código muerto), no bypass del allowlist.
- **Service key nunca interpolada en errores** (`writer-supabase.ts:13`, guarda T-06-06).
- **Audit inmutable** (`identidad_audit_immutable`, 0006) y **DLQ pgmq** (`ingest_dlq`, 0003) operativos.
- **Degradación honesta de texto-fuente:** ausencia de texto → `idea_matriz` null, no fabricación.
