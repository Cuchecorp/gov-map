## Spacing Scale

Project uses Tailwind default 4px scale. Bento-specific values below. Declared values are multiples of 4 **except two intentional off-step exceptions carried from the mockup grid geometry** (documented, not a violation).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/inline gaps (`gap-1`) |
| sm | 8px | Compact spacing, pill gap (`gap-2`) |
| md | 16px | Default element spacing (`gap-4`) |
| lg | 24px | Container horizontal padding (`px-6`), header/footer padding |
| xl | 32px | Section breaks |
| 2xl | 48px | Major section breaks |

**Intentional exceptions (off-step, from mockup — do NOT round):**

- **BentoGrid gap: `14px`** (not 12/16). This is the mockup's grid gutter and the primitive's defining constant. Implement as `gap-[14px]` (Tailwind arbitrary) or a CSS var `--bento-gap: 14px`. Rounding to 16 changes tile proportions across the whole grid — banned.
- **`--radius-control: 11px`** — mockup input/button radius. Off the shadcn `--radius` (8px) step by design; used only by controls inside bento contexts (SearchBox input/button in Phase 77). Precedent: `.net-chip` 11px off-step is already a repo-sanctioned exception (DEBT-05).

**Header/footer padding (chrome, this phase):**
- Container: `max-w-[1120px] mx-auto`.
- Header inner padding: `px-6 py-3` (24px / 12px) — matches mockup `padding:12px 24px`.
- Footer: `mt-16 pt-6` above content (mockup `margin-top:40px; padding-top:24px` ≈ `mt-10 pt-6`; keep repo's existing `mt-16` unless it visibly regresses — spacing is Claude's discretion here, `mt-10`/`mt-16` both acceptable, pick one and fix it in the plan).
- `main` container (only where it does not break internal layout): `max-w-[1120px] px-6 mx-auto`.

## Typography

Geist / Geist Mono, already loaded. This phase renders NO new page body copy beyond the footer (unchanged) — the type roles below are the **primitive contract** the tiles and chrome consume. Weights: exactly two — 400 (regular) and 600 (semibold). Mono weight 500 permitted for kicker/tally/dates (Geist Mono metadata role only).

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Wordmark (header) | 16px | 600 | 1.1 (`leading-tight`) | `tracking-tight` (-0.01em), Geist sans |
| Nav item | 14px (`text-sm`) | 500 | 1 (inline-flex) | existing HeaderNav — unchanged |
| Tile heading (h2) | 18px–21px | 600 | 1.15–1.2 | primitive default for `BentoTile` heading slot; exact per-tile size set in 77/78 |
| Body / tile text | 14px (`text-sm`) | 400 | 1.5 | `text-muted-foreground` for secondary |
| Footer body | 13px | 400 | 1.6 | existing footer — unchanged |
| Kicker (mono) | 11px | 500 | 1 | Geist Mono, `uppercase`, `tracking-[0.08em]`, `text-muted-foreground` — primitive available this phase, consumed by hero in 77 |
| Metadata (mono) | 12px | 400/500 | 1.5 | Geist Mono, dates/tallies — consumed in 78 |

**Line-height defaults for the primitive:** body 1.5, headings 1.2. Headings ≥28px (hero h1 in 77) drop to 1.1.

**scroll-margin-top on anchor headings (this phase — risk mitigation #2):** sticky header will occlude in-page anchors on long fichas. Add a global `scroll-margin-top` to heading elements that act as scroll anchors (`h1,h2,h3[id]` or the ficha-rail anchor targets) equal to header height + breathing room: **`scroll-mt-20` (80px)** or a token `--header-offset: 72px` + margin. Set once, globally, in `globals.css` or layout; verify skip-link target and ficha-rail anchors land below the sticky header.

## Color

Light-only surface (dark theme derived, not primary — invariant 8). The 60/30/10 split maps cleanly onto existing tokens:

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (crema `40 33% 97%`) | Page surface, header/footer background, input fill inside bento |
| Secondary (30%) | `--card` (`40 30% 99%`) + `--border` (`40 16% 86%`) | BentoTile `default` fill + hairline border; footer border-top |
| Accent (10%) | `--accent-product` (petróleo `183 38% 26%`) | see reserved-for list below |
| Accent-soft | `--accent-product-soft` (`hsl(183 30% 93%)`) | tile-acento internal chips, urgencia pill bg (Phase 78) |
| Destructive | `--destructive` (`0 72% 42%`) | **not used this phase** (no destructive actions) |

**Accent (`--accent-product`) reserved for — explicit list (this phase's primitives):**
1. BrandIcon stroke/fill in the wordmark and the `accent` tile.
2. `BentoTile` variant `accent` background (inverted teal tile) + its foreground text.
3. Focus rings (`--ring` is already `183 38% 26%` = petróleo) — `focus-visible` outline on tile-links and controls.
4. Link/CTA text inside tiles (`text-accent-product`), active nav underline (existing).
5. Hover border on interactive tiles (`hover:border-accent-product`).

Accent is **never** a tile fill for `default` tiles, never a data-bearing color (Cámara/Senado bars use civic tokens, arriving Phase 78), and never signals affinity/proximity (anti-insinuación).

**BentoTile `accent` variant — contrast (invariant 7, WCAG AA):** inverted teal tile has white/near-white text on `--accent-product`. Body text on the accent tile must hit ≥4.5:1. The mockup's `rgba(234,242,241,0.85)` body on `#2A5859` must be verified ≥4.5:1; if it fails, raise opacity to full or lighten toward `#EAF2F1` at full alpha. Heading (`#FFFFFF`) is fine. Do this by deriving from tokens, not hardcoding the rgba.

**Dark pair:** `--accent-product` dark is already `183 34% 46%`; `--accent-product-soft` dark is `hsl(183 30% 18%)`; `--card`/`--border`/`--background` all have dark values. BentoTile default and accent derive their dark appearance from these existing tokens — no new dark tokens needed. New radius tokens (`--radius-tile`, `--radius-control`) are theme-agnostic (no dark pair required). Full dark verification is Phase 80; this phase only ensures the primitive references dark-aware tokens.

**New tokens introduced this phase (globals.css `:root`, per D4):**

```css
--radius-tile: 16px;    /* bento tile corner; --radius (shadcn 0.5rem) untouched */
--radius-control: 11px; /* bento input/button corner; off-step, mockup-exact */
```

The token comment must document the mockup→token color map (above) so the cardinal rule is self-evident at the definition site. `--radius` (shadcn 8px) is **NOT** touched — zero shape regression on interior routes.
