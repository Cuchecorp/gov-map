# Phase 57 Plan 05: R2 wiring en prod CLI de leyes + source_snapshot registry

## Objective

Cerrar el gap G23/CRON-02: el cron `leyes-weekly.yml` invoca `run-tramitacion-prod-cli.ts`
que **nunca escribe R2** aunque `ingest-run.ts` ya acepta `opts.r2Store`. Además, ningún
conector de tramitación registra filas `source_snapshot` → la señal `r2Snapshot` del CLI
de frescura permanece "n/d" para "leyes".

## Context

- `ingest-run.ts` L269–291 ya tiene Etapa-1 R2 completa (putImmutable + skip si existed).
- `ingest-cli.ts` construye R2Store desde env y lo pasa a `runIngest`.
- `run-tramitacion-prod-cli.ts` llama `ingestMain()` (= `main()` de ingest-cli) → YA recibe
  el R2Store construido allí. El gap real: las env vars R2_* no se propagan al `loadEnv()` de
  run-tramitacion-prod-cli.ts (solo carga SUPABASE_*). Fix: ampliar `loadEnv` para también
  exponer R2_* al process.env antes de que ingest-cli.main() las lea.
- `source_snapshot`: el registro debe hacerse en `ingest-run.ts` donde se hace putImmutable,
  inyectando un `snapshotWriter` opcional (igual que probidad). run-tramitacion-prod-cli.ts
  lo construye solo LIVE con creds Supabase.
- Fuente a registrar: `"leyes"` (exacto — coincide con catalog.ts fuente="leyes" y la query
  `WHERE source = 'leyes'` de query-runner.ts).

## Tasks

### Task 1 — Wire R2 env vars + snapshotWriter en run-tramitacion-prod-cli.ts

**Type:** auto
**Files:** `packages/tramitacion/src/run-tramitacion-prod-cli.ts`

1. Ampliar `loadEnv()` para leer `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_ENDPOINT_URL`,
   `R2_BUCKET` además de las SUPABASE_*.
2. Exponer las vars R2_* al `process.env` antes de llamar `ingestMain()` (ingest-cli.main
   construye R2Store desde `process.env.R2_ACCESS_KEY_ID`).
3. Construir un `snapshotWriter` (SnapshotWriter + SupabaseSnapshotStore + createClient) solo
   LIVE con creds Supabase — espejo exacto del patrón en run-probidad-todos-cli.ts L126-137.
4. Pasar `snapshotWriter` a `ingestMain()` (que lo acepta como `IngestCliOptions.snapshotWriter`
   — agregar el campo si no existe; ver tarea 2).
5. WARN loud cuando R2 no configurado (mismo mensaje que ingest-cli: "[WARN] R2 no configurado").

### Task 2 — Agregar snapshotWriter a IngestCliOptions y pasarlo a runIngest

**Type:** auto
**Files:**
  - `packages/tramitacion/src/ingest-cli.ts`
  - `packages/tramitacion/src/ingest-run.ts`

1. Agregar `snapshotWriter?: SnapshotWriter` a `IngestCliOptions` (ingest-cli.ts).
2. Pasar `opts.snapshotWriter` al `runIngest(...)` call (ingest-cli.ts L283-293).
3. Agregar `snapshotWriter?: SnapshotWriter` a `RunIngestOpts` (ingest-run.ts).
4. En `runIngest`, tras el putImmutable exitoso (L275-287), si `opts.snapshotWriter` está
   presente, llamar `snapshotWriter.write({...})` best-effort (try/catch no fatal) con:
   - `source: "leyes"` (exacto)
   - `resource: boletinFull`
   - `cacheKey: sha`
   - `r2Path`
   - `contentHash: sha`
   - `fingerprint: sha.slice(0, 8)`
   - `dateBucket: today`
   - `provenance: { sourceUrl: "https://tramitacion.senado.cl/", fetchedAt: new Date().toISOString() }`
