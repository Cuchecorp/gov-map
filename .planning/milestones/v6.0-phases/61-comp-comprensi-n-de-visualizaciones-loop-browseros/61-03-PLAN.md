### Task 3 — Capturas móvil (390px) de todas las superficies
type: auto

Para cada superficie (mismas 7), abrir hidden page, setear viewport 390×844 vía `evaluate_script` (`window.resizeTo` o `meta viewport`), esperar 5s, screenshot a `comp-evidence/<superficie>-mobile-antes.png`.

Alternativamente: usar parámetro de resize vía `evaluate_script` si el tool no soporta viewport nativo:
```js
Object.defineProperty(window, 'innerWidth', {value: 390});
```

**Done:** 7 screenshots móvil en comp-evidence.

### Task 4 — Lectura fría + audit 61-COMP-AUDIT.md
type: auto

Leer cada screenshot con mentalidad cero-contexto. Para cada imagen responder:
- ¿Qué entiende alguien sin saber qué es gov-map / Observatorio?
- ¿Qué pregunta responde cada visualización?
- ¿Qué número/serie/elemento no se entiende?
- ¿La sección de cruces se entiende sin contexto previo?

Veredicto por superficie: comprensible | mejorable(P1) | no-se-entiende(P0).

Escribir `.planning/phases/61-comp-comprensi-n-de-visualizaciones-loop-browseros/61-COMP-AUDIT.md` con tabla:

| ID | Superficie | Viewport | Severidad | Qué confunde | Fix propuesto | Screenshot |
|----|-----------|----------|-----------|--------------|---------------|-----------|

Regla: si la leyenda de cruces deja CUALQUIER confusión, mínimo P1.

**Done:** 61-COMP-AUDIT.md escrito con todos los hallazgos.
